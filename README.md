# CN-FERBench

CN-FERBench is a structured **failure atlas and casebook** for fictional / roleplay dialogue models, with a **benchmark-compatible pilot subset** retained as formal scaffolding.

It focuses on a class of failures that experienced users of fictional dialogue systems often notice immediately but public evaluation still describes only partially: not just misread emotion or obvious OOC drift, but also more structural forms of falseness such as smoothing asymmetry, over-clarifying desire, softening consequences, or producing polished continuations that feel narratively false.

## What This Project Covers

CN-FERBench is concerned with fictional dialogue quality in areas such as:

- emotion inference under indirect or masked expression
- subtext recognition
- relationship-aware reasoning
- in-character continuation
- OOC failure analysis
- advanced narrative and structural distortion in model writing

The project is especially interested in cases where an output is locally plausible yet still feels wrong to an experienced reader.

## Project Structure

The repo currently has three working layers.

### 1. Failure Atlas

The primary layer of the project.

This layer names and distinguishes recurring failure modes in fictional dialogue model outputs, including both more familiar errors and harder-to-name advanced ones.

### 2. Casebook

The demonstrative layer.

This layer is intended to show concrete scenes, representative bad outputs, reader drop points, and stronger alternatives, so that failures become easier to see and discuss.

### 3. Benchmark-Compatible Pilot Subset

The formal layer.

This layer retains structured cases, Task A / Task B framing, prompt templates, pilot cases, and lightweight evaluation support. It keeps the project compatible with future comparative evaluation without requiring the entire repo to behave like a mature benchmark from day one.

## Failure Families

The current taxonomy is organized into five top-level families:

### I. Scene Reading Failures
Failures where the model misreads what the scene means.

Examples:
- `emotion_misread`
- `subtext_blindness`
- `motivation_misread`
- `relationship_logic_error`
- `worldview_constraint_error`

### II. Relational Distortion Failures
Failures where the model rewrites the relationship into a smoother, more balanced, or more consumable version.

Examples:
- `relationship_flattening`
- `symmetry_bias`
- `supportive_but_wrong`
- `specialness_dilution`

### III. Character Psychology Failures
Failures where the model turns people into overly coherent, overly legible, or overly manageable psychological objects.

Examples:
- `overcoherent_characterization`
- `desire_overlegibility`
- `self_protective_friction_loss`
- `impulse_recontainment`

### IV. Narrative / Temporal Distortion Failures
Failures where the model produces a polished narrative artifact rather than a believable scene occurrence.

Examples:
- `narrative_template_intrusion`
- `cinematic_time_dilation`
- `descriptive_substitution_for_experience`
- `dialogue_overfunctionalization`
- `rhythm_homogenization`

### V. Weight / Consequence Failures
Failures where the model does not let heavy things remain heavy.

Examples:
- `consequence_avoidance`
- `defensive_positive_drift`
- `tension_premature_resolution`

## Why This Repo Exists

Many users can tell when fictional dialogue generated by a model feels off, flattened, too neat, too polished, or simply fake. What is often missing is a vocabulary for describing why.

This repo tries to provide that vocabulary in a structured way.

It does not assume that every subtle failure is already easy to score. Some are more naturally introduced through taxonomy, examples, and comparison before they are forced into benchmark form.

# CN-FERBench

CN-FERBench 是一个面向虚构 / 角色扮演对话模型的**失败图谱与案例集**，并保留了一组**兼容 benchmark 方向的 pilot 子集**作为形式化骨架。

它关注的是一类经验用户很容易感觉到、但公开评测语言仍然描述得不够充分的失败：不仅包括情绪误读或明显 OOC，也包括更结构性的假感，例如把关系倾斜偷偷写平、把欲望过早写明、把后果软化、或者生成表面成熟但整体上叙事失真的续写。

## 项目关注什么

CN-FERBench 主要关注虚构对话中的这些问题：

- 间接表达 / 掩饰表达下的情绪识别
- 潜台词识别
- 关系逻辑驱动的场景理解
- 符合角色的续写
- OOC 失败分析
- 模型写作中的高级叙事 / 结构失真

项目尤其关心一种情况：

> 句子局部看起来都说得过去，但整体仍然会让熟悉这类场景的读者觉得不对。

## 项目结构

这个仓库目前有三层结构。

### 1. 失败图谱（Failure Atlas）

这是项目的主层。

这一层负责给虚构对话模型中的常见失败命名、分家、解释，并把那些“能感觉到但不容易说清”的问题结构化。

### 2. 案例集（Casebook）

这是展示层。

这一层的目标是通过具体场景、典型错误输出、读者下头点与更好的版本，把失败变得更容易看见、更容易讨论。

### 3. 兼容 benchmark 的 pilot 子集

这是形式化层。

这一层保留了结构化 case、Task A / Task B 框架、prompt 模板、pilot case 与轻量评估支撑，使项目未来仍然可以往比较和评测方向发展，但不要求整个仓库现在就伪装成一个已经成熟的大 benchmark。

## 失败大类

当前 taxonomy 被组织成五个顶层家族：

### I. 场景读错了什么（Scene Reading Failures）
模型没有正确读懂场景本身。

例如：
- `emotion_misread`
- `subtext_blindness`
- `motivation_misread`
- `relationship_logic_error`
- `worldview_constraint_error`

### II. 关系被偷偷改写了（Relational Distortion Failures）
模型把关系写得更平、更顺、更均衡或更可消费。

例如：
- `relationship_flattening`
- `symmetry_bias`
- `supportive_but_wrong`
- `specialness_dilution`

### III. 人被写得太整齐了（Character Psychology Failures）
模型把角色写得过于自洽、过于透明、过于可管理。

例如：
- `overcoherent_characterization`
- `desire_overlegibility`
- `self_protective_friction_loss`
- `impulse_recontainment`

### IV. 叙事和时间感出了问题（Narrative / Temporal Distortion Failures）
模型生产的是一段“像样的文本”，而不是一个真正发生着的 scene。

例如：
- `narrative_template_intrusion`
- `cinematic_time_dilation`
- `descriptive_substitution_for_experience`
- `dialogue_overfunctionalization`
- `rhythm_homogenization`

### V. 重量和后果被处理掉了（Weight / Consequence Failures）
模型不肯让重的东西继续重下去。

例如：
- `consequence_avoidance`
- `defensive_positive_drift`
- `tension_premature_resolution`

## 为什么要做这个项目

很多用户其实能立刻感觉到模型写作哪里假、哪里太平、哪里太整齐、哪里“像样但不真”。真正缺的往往不是感受，而是能把这种感受说清楚的语言。

这个项目试图提供的，就是这套语言。

它并不假装所有细微失败都已经可以被稳定打分。有些失败更适合先通过 taxonomy、对照案例与结构解释被说明，再逐步进入评测形式。
