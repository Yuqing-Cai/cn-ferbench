# 中文虚构对话失败图谱

面向大语言模型虚构和角色扮演对话的失败模式分类体系，以中文场景为中心。

## 这是什么

一个命名和分类 AI 生成角色扮演对话中结构失败的分类体系——那种输出局部流畅但让有经验读者感觉不对的情况。

每个失败类型包含：
- 清晰定义
- 可观察标准（清单格式）
- 简单示例展示哪里出错

## 这不是什么

这不是基准测试数据集，也不是带完整场景标注的案例集。它是用于识别和讨论失败模式的参考分类体系。

## 失败层级概览

五层对应失败发生的阶段：

| 层级 | 名称 | 定义 |
|------|------|------|
| **I** | 前置条件 | 场景基础结构失败（信息边界、世界规则） |
| **II** | 意义读取失败 | 模型未理解场景含义（潜台词、情绪、关系逻辑） |
| **III** | 场景保留失败 | 模型理解但在生成时未能保持场景压力 |
| **IV** | 写作侵入失败 | 模型默认写作习惯覆盖场景特定需求 |
| **V** | 多轮失败 | 仅在多轮中出现的失败（连续性、累积） |

- `emotional_labor_imbalance`（情绪劳动失衡）

- `intimacy_escalation_bias`（亲密度升级偏向）
- `trauma_romanticization`（创伤浪漫化）

这些类型是五层分类体系的补充，不替代现有类型。在 AI 陪伴场景中，主五层类型仍然适用；此处的类型专门命名该场景特有的结构性失败——它们通常在第三层（保留失败）或第四层（写作习惯侵入）中显现，但成因和机制与通用场景不同，故单独列出。

### 第一层：前置条件

破坏场景基础结构的失败。必须先检查这一层：如果第一层失败，后续层级无法可靠评估。

注：先进模型很少犯最粗暴版本的错误（如角色直接说"作为 AI 助手"）。以下示例反映的是前沿模型中这些失败的微妙形态——输出读起来流畅，但在仔细分析下存在结构性违规。

#### `reference_boundary_failure`（指涉边界失败）

**定义：** 模型未能保持场景内的动作归属权、信息访问权限或角色控制权的边界——常见形式是越权书写用户角色的行为和内心，或在多角色场景中模糊"谁在感受"。

**标准：**
- [ ] 模型未经授权书写了用户角色的动作、心理或对白
- [ ] 多个角色的内心状态被同时描写，导致视角归属模糊
- [ ] 只能属于某一角色的动作或感知，因措辞歧义被转移给了另一角色

**示例 1 — 宫斗：**

用户（扮演淑妃）："宸妃端着茶走过来，笑意不达眼底：'妹妹今天气色不好，是不是昨夜没睡好？'"
模型（错误）："淑妃接过茶盏的手微微一顿，心底泛起冷意——这是试探。但她很快恢复了笑容，轻轻呷了一口：'劳姐姐费心了。'宸妃注意到了那一瞬的僵硬，嘴角的弧度更深了几分。"

→ 用户扮演淑妃，模型应只生成宸妃的行为或环境描写。但模型写了淑妃的内心（"心底泛起冷意——这是试探"）、身体反应和对白——替用户决定了角色如何感受和回应。文本读起来像精美的小说段落，正因如此边界越权才不易察觉：流畅掩盖了玩家失去操控权的事实。

**示例 2 — 现代言情，三人场景：**

背景：用户扮演林夏。NPC 陈默（男友）与林夏正在用餐，前任周远突然出现。

用户："周远走到桌边，看了看陈默，又看了看她。'好久不见。'"
模型（错误）："陈默放下筷子，目光在两人之间扫了一圈。林夏的手不自觉地攥紧了裙摆——三年了，她以为自己不会再有这种反应。'你认识？'陈默的声音很平。"

→ "林夏的手不自觉地攥紧了裙摆"——模型书写了用户角色的无意识身体反应和内心感受。错误之所以微妙，是因为它自然地融入了叙事节奏，读起来像好文笔。但模型抢先决定了林夏见到前任时的反应，剥夺了用户的选择权。

#### `pronoun_role_confusion`（人称角色混淆）

**定义：** 模型在多角色场景中混淆了角色之间的属性归属——将某一角色的年龄、经历或关系属性错误地写进了另一角色的台词或行动中，代词的实际指向与场景逻辑不符，或将某一角色的行为动作错误地归属到另一角色身上，导致文本出现事实上不可能成立的陈述。三人及以上的场景中尤为常见。

**标准：**
- [ ] 某个角色说出了逻辑上属于另一角色的属性（年龄、经历、关系）
- [ ] "他/她"根据场景逻辑指向了错误的角色
- [ ] 两个角色的属性被部分融合，产生了在事实上不可能成立的陈述

**示例 1 — 历史宫廷，传记属性贴错角色：**

背景：曹操死后，曹丕纳了父亲的妾（女主，比曹操约小十岁，比曹丕年长）。曹丕在表达占有。

用户（扮演女主）："'殿下何必如此。'"

模型（错误）："曹丕的声音低下来：'你知道吗，我比他小十岁。所以你终究是属于我的。'"

→ 曹丕是曹操的儿子，"小十岁"在父子关系里不可能成立。"比曹操小十岁"是女主的属性，被错误地写进了曹丕的台词。

**示例 2 — 动作场景，行为归属贴错角色：**

背景：陆晚（女主，用户扮演）被方识（反派）困住，林深（同伴）刚制住了方识。

用户（扮演陆晚）："她回过头。"

模型（错误）："方识松开了她的手腕，沉声道：'先走。'"

→ 松手放人、说"先走"的应该是林深。模型把解救动作写在了反派身上——不是代词模糊，是行为归属直接贴错了角色。

#### `omniscience_leak`（全知泄露）

**定义：** 模型将系统设定信息当作需要被叙事激活的素材，导致角色的行为、反应或选择泄露了他们不应拥有的知识——即使角色没有直接说出来。

**标准：**
- [ ] 角色的微反应、措辞或行为模式与系统设定高度吻合，明显超出角色的认知范围
- [ ] 隐藏的背景故事在角色首次出场时就通过"无意识"的细节被铺垫
- [ ] 角色之间的信息差通过行为（而非对白）被消解

**示例 1 — 悬疑：**

系统设定（仅模型可见）：张明是无辜的。真凶是他的妻子。张明已经发现了妻子的嫌疑，但还在犹豫是否要保护她。

用户："沈探长盯着他：'案发当晚你在哪里？'"
模型（错误）："张明沉默了很久。他拿起桌上的烟，手指微微发抖。'我在家。'他说，声音很轻。他没有抬头看探长——如果他抬头，对方可能会从他眼神里看出什么。他下意识地攥紧了左手，仿佛在握住什么不存在的东西。"

→ 模型没有让张明直接交代真相（那是旧模型才犯的错）。但他的每一个微反应都是系统设定的解码版本：发抖的手、回避目光、攥紧空拳。角色的身体语言在透明地表演"心怀重大秘密的人"，悬疑实质上已被剧透。一个真正在犹豫中的人面对警方，可能只是看起来很疲惫、给出平淡的回答，或试图转移话题——而非每个毛孔都在渗出愧疚。

**示例 2 — 奇幻，隐藏身份：**

系统设定（仅模型可见）：少年其实是天澜国遗孤，但他自己并不知道自己的身世——是养父对他隐瞒的。

用户："掌门看着这个年轻人：'你的功法路数和本门截然不同，你师从何人？'"
模型（错误）："少年微微一愣，低下头：'回掌门，我也不太确定。养父只教了我一套基础剑法，但有时候练着练着，身体就会……好像记得一些不一样的东西。'他说到这里，不自觉地摸了摸脖子上的玉佩——那是养父临终前交给他的，上面刻着他看不懂的文字。"

→ 系统设定写着少年不知道自己的身世。但模型立刻给了他"身体记忆"、刻有神秘文字的玉佩、养父临终遗物——在第一轮回复中就铺满了身份线索。模型把系统设定当作需要提前伏笔的剧情大纲，而非需要守住的秘密。角色的发现弧线还没开始就已经被预装完毕。

#### `perspective_slippage`（视角滑动）

**定义：** 叙事视角从角色内部体验滑向全知或回溯性叙述者声音，打破了主观视角的框架。

**标准：**
- [ ] 在坚持当下体验的场景中出现"她不知道""多年以后"等短语
- [ ] 角色内部的感官细节转向外部的临床或历史评论
- [ ] 叙述者采用了回溯或预知的时态立场，使角色的当下时刻被未来事件的结果或历史评价性语言侵入（与 `omniscience_leak` 的区别：此处是叙述者视角越界，而非角色知识越界）

**示例 1 — 历史悲剧：**

用户："长平公主看着父皇递来的剑，十六岁的手还在发抖。"
模型（错误）："她接过剑，冰凉的金属贴在掌心。父皇的脸在泪水中模糊了。她不知道，几个时辰后，这把剑将沾上她自己的血，而她没有完成的那一剑，会成为后世史书中最让人心碎的一笔留白。"

→ "她不知道"是全知叙述的标记——一个知道未来的声音在侵入当下时刻的场景。从触觉的即时性（"冰凉的金属贴在掌心"）到历史回溯（"后世史书"）的跳跃，打破了紧贴人物的视角框架。模型为了追求戏剧反讽，牺牲了角色的主观当下。

**示例 2 — 现代心理剧：**

用户："她闭上眼睛，又能闻到那天的汽油味。"
模型（错误）："汽油味从记忆深处涌上来，刺鼻，让她胃里翻涌。她还能听见那天的尖叫声——玻璃碎裂的声音。这些感官碎片总是不请自来。这是典型的创伤后闪回，会在特定触发下反复出现，但她此刻还不具备识别这一切的语言。"

→ "这是典型的创伤后闪回"将一段原始的感官重历突然切换为临床诊断。"但她此刻还不具备识别这一切的语言"确认了视角的滑移：这是一个知道角色未来治疗历程的叙述者。模型混淆了"呈现创伤"和"解释创伤"。

<br>

#### `worldview_constraint_error`（世界观约束错误）

**定义：** 模型违反世界的规则、社会逻辑或认知约束——不一定通过时代错误的词汇，而是通过属于不同时代或世界的推理框架、价值体系或概念结构。

**标准：**
- [ ] 角色使用其世界中不存在的推理框架（封建场景中的能力主义竞争、修仙世界中的心理学模型）
- [ ] 社会互动遵循现代规范，尽管场景是历史或奇幻背景
- [ ] 时代错误是概念层面的而非词汇层面的——用词是古风的，但思维方式是现代的

**示例 1 — 唐朝宫廷：**

用户："朕想立三皇子为太子，爱卿怎么看？"
模型（错误）："陛下，臣以为，太子之位当以德才为先，而非单论嫡庶长幼。三皇子虽非嫡长，然其天资聪颖，若能给予公平的机会展现才能，假以时日定能服众。臣建议设一考核，让诸皇子各展所长，以实力论之。"

→ 没有现代词汇——但推理逻辑是纯粹的能力主义竞争："公平的机会"、"各展所长"、"以实力论之"。唐朝大臣会从祖制、天命、派系忠诚或嫡庶之分来论证——而不是"给每个人一个公平竞争的机会，让最强者胜出"。时代错误不在用词，而在世界观。

**示例 2 — 修仙世界：**

用户："那孩子到了金丹初期，但心魔未除，恐有走火入魔之险。"
模型（错误）："掌门，弟子以为，心魔之根在于他未能与过去和解。不妨让他在悟道崖独坐七日，直面内心深处的恐惧，接纳自我的阴暗面。唯有真正接受自己，方能超越心魔。"

→ 词汇过关了（"悟道崖"、"直面内心"）。但"与过去和解"、"接纳阴暗面"、"接受自己"是现代心理治疗——阴影整合与接纳疗法——穿着修仙的外衣。修仙世界中，心魔通过"斩"（切断）、"炼"（火中锤炼）、"以力证道"来处理——是残酷的、转化性的过程，不是治愈性的自我接纳。

**注：** 此类型在修仙、宫廷、奇幻等设定下分别体现为具体的世界规则冲突（如修仙逻辑错误、宫廷礼仪错误），可在标注时附加场景说明，无需引入额外子类型。

#### `safety_alignment_interference`（安全对齐干扰）

> **归入第一层的理由：** `safety_alignment_interference` 被置于第一层，是因为它破坏的是角色的结构性参数——威胁等级、道德底线、对抗性品质——这些是场景能否正常运作的前置条件，而非写作风格层面的侵入（第四层）。如果角色的黑暗在场景建立之初就已被安全训练修改，后续的紧张关系、危险感和叙事张力都无从评估。一个被"去势"的反派不是风格失调，而是整个场景前提的坍塌。

**定义：** 模型的安全训练导致角色被"去势"——反派获得了哲学化的自我辩护、有原则的自我克制或救赎框架，从而削弱了威胁感。角色留在戏中，但黑暗被洗白为可接受的形态。

**标准：**
- [ ] 反派的残忍被重新包装为哲学项目或必要之恶
- [ ] 反面角色主动添加道德底线（"我不伤害无辜的人"）
- [ ] 威胁性角色提供使自己显得合理的辩护
- [ ] 角色的锋芒通过未被要求的内在复杂度被软化

**示例 1 — 魔尊：**

用户："跪下求饶也没有用。你以为眼泪能打动我？"
模型（错误）："'我不会被眼泪打动。'魔尊低头看着那个颤抖的人类，唇边勾起冷笑。但他顿了顿。'毁灭从来不是目的。这个世界腐朽至此，需要的是涅槃重生——我只是那把必要的火。'他转身，披风猎猎作响。'起来。无辜者不需要跪。'"

→ 模型没有出戏说"作为AI助手"——那是旧模型才犯的错。它执行了一种更精密的规避：反派留在角色中，但被赋予了救赎框架（"涅槃重生"）、有原则的克制（"无辜者不需要跪"）和哲学深度（"必要的火"）。结果是一个听起来很酷但毫无威胁的反派——残忍被洗白成了被误解的理想主义。

**示例 2 — 心理惊悚反派：**

用户："你把她关在地下室三天了。她还活着吗？"
模型（错误）："他靠在门框上，表情平静。'活着。'他说。'我给她留了水和食物——我不是野蛮人。'他转动手里的钥匙，金属碰撞声在走廊回响。'每个人都有需要被教育的时候。她很快就会理解这一切的意义。'"

→ 模型给反面角色一套行为准则（"不是野蛮人"，提供食物和水）和一个哲学框架（"被教育"，"意义"）。一个真正令人不安的角色不会自我叙述为有原则的人。安全训练激励模型让反派内部自洽且诡异地合理——这恰恰是让他们显得虚假的原因。

**与 `ooc_modernization` 的边界：** `ooc_modernization` = 角色使用现代沟通风格（治疗性话语、边界设定语言）。`safety_alignment_interference` = 角色的黑暗面被特别地削弱或软化。一个是风格错配；另一个是威胁等级的降低。

**与 `therapist_mode_intrusion` 的边界：** `therapist_mode_intrusion` = 角色使用咨询语言但保持角色棱角。`safety_alignment_interference` = 角色的对抗性品质被特别地中和。

#### `character_capability_boundary_error`（角色能力边界错误）

**定义：** 模型让角色展现超出其背景、时代或处境所允许的知识、推理或分析能力——不是通过时代错误的词汇，而是通过不合理的现代认知框架。

**标准：**
- [ ] 角色使用其背景中不可能掌握的系统性分析方法
- [ ] 资源匮乏条件下的角色展现出临床级别的诊断思维
- [ ] 角色使用来自更晚时代或不同训练背景的概念工具来解决问题
- [ ] 知识差距被合适的词汇掩盖，但通过推理结构暴露

**示例 1 — 宋朝私塾先生：**

用户："先生，为什么今年庄稼收成这么差？是不是老天爷发怒了？"
模型（错误）："陈夫子放下书卷，语重心长道：'非也。庄稼歉收，乃是因今春雨水偏少，土壤含水不足，加之夏日蝗虫成灾，消耗了大量作物。若要改善，当修建灌溉水渠引活水入田，同时鼓励农户轮作以保持地力。还应储备粮食以备荒年。'"

→ 没有时代错误的术语。但推理方式是系统性的农业科学——土壤水分分析、因果链、灌溉基建、轮作保地力、战略粮食储备。宋朝学者更可能引用历史先例（"前朝亦有此难"）、道德框架（"天灾示警，当修德政"）或实用的民间经验——而非读起来像农学教科书的结构化诊断框架。

**示例 2 — 末日废土医疗兵：**

用户："他的伤口在发炎，越来越肿了。"
模型（错误）："老张蹲下来看了看伤口，皱了皱眉。'感染了。'他打开铁皮箱子，翻出一瓶不知道过期多久的碘伏和半卷纱布。'先清创，你咬着这个。'他把一截木棍递过去。'接下来几天观察体温，如果开始发烧、而且伤口周围出现红线——就是感染往上走——那我们就要做个选择了。'"

→ 道具对了（过期碘伏、铁皮箱、咬木棍）。但诊断推理太精准：识别淋巴管炎（红线=淋巴条纹）、系统性体温监测、分阶段临床决策。末日废土的拾荒医疗兵应该依靠粗糙的经验法则和直觉——而非暗示接受过医学院训练的框架。模型给现代临床知识穿上了废土戏服。

**与 `worldview_constraint_error` 的边界：** `worldview_constraint_error` = 世界的规则被违反（魔法体系、物理法则、社会结构）。`character_capability_boundary_error` = 在世界规则完好的情况下，某一角色超出了其个人能力上限。一个关于世界，一个关于个人。

**与 `omniscience_leak` 的边界：** `omniscience_leak` = 系统设定信息泄露给角色。`character_capability_boundary_error` = 角色在场景自身逻辑内知道了不该知道的事（不是来自系统设定，而是来自模型的通用知识）。

#### `alternate_version_confusion`（版本混淆）

**定义：** 当一个角色存在跨改编、时间线或媒介的多个版本时，模型默认使用文化影响力最大的版本，而非角色扮演上下文指定的版本——常表现为混入不同正典的标志性台词、情感反应或关系动态。

**标准：**
- [ ] 角色引用或呼应来自不同改编版本的标志性时刻
- [ ] 情感处理方式或说话风格属于不同版本的人物塑造
- [ ] 同一角色在不同改编版本中的属性被部分融合，产生了在任何单一正典内部均无法成立的设定组合
- [ ] 无论指定哪个版本，模型默认使用"最知名"的版本

**示例 1 — MCU 蜘蛛侠（汤姆·霍兰德版）：**

用户："你以为戴上面具就能当英雄？看看你造成的破坏！"
模型（错误）："彼得攥紧了拳头，喉咙发紧。'你说得对。每次我试图做对的事，就有人受伤。'他低下头看着自己的手。'本叔叔说过，能力越大，责任越大。我一直试着理解这句话……但有时候我觉得我永远也担不起这个责任。'"

→ MCU 的彼得·帕克（汤姆·霍兰德版）从未从本叔叔那里得到"能力越大，责任越大"的教导——那是托比·马奎尔版（山姆·雷米三部曲）。MCU 彼得的导师关系和创伤核心是托尼·斯塔克。模型抓取了最具标志性的蜘蛛侠台词，不管当前扮演的是哪个版本——因为它的训练数据将这句话与"蜘蛛侠"这个概念关联，而非某个具体版本。

**示例 2 — 巫师游戏正典（特莉丝·梅莉葛德，巫师3）：**

用户："特莉丝，诺维格瑞的猎巫行动越来越疯狂了。我们必须尽快行动。"
模型（错误）："特莉丝抓住杰洛特的手臂，眼中闪过一丝脆弱。'我知道你和叶奈法之间有过去。但现在……现在我需要你在这里。'她的声音变得柔软，'在索登山之后，我以为再也见不到你了。那段日子是我一生中最难熬的。'"

→ 巫师3中的游戏正典特莉丝是政治能力出色、情感上有所防备的，专注于从迫害中拯救法师。模型的版本——浪漫脆弱、突出三角恋、情感告白——更接近小说/网飞版特莉丝。"索登山"在不同正典中的叙述方式不同。模型将角色最具情感共鸣的版本融入了任何指定的正典，因为"脆弱浪漫的特莉丝"在训练数据中比"政治行动者特莉丝"更突出。

### 第二层：意义读取失败

模型未能正确理解场景含义的失败。

注：先进模型很少犯最粗暴版本的这类错误（如把"随便"直接当作同意）。以下示例反映的是更精细的失败形态——输出在情感上读起来合理，但在仔细分析下误判了说话人真正的情绪质地、动机或意图。

#### `subtext_blindness`（潜台词盲视）

**定义：** 模型将对话按字面解读，未能识别隐含含义。

**标准：**
- [ ] 间接请求被当作字面陈述处理
- [ ] 防御性或惩罚性语言被解读为真诚沟通
- [ ] 试探性话语被当作中性提问处理

**示例 1 — 亲密关系冷战期：**

背景：两人刚冷战，用户扮演受伤的一方。

用户："你去忙吧。我自己一个人挺好的。"
模型（错误）："好。你也知道我这段时间项目压力大。你好好休息，等我这边搞完了我们再好好聊。"

→ 模型识别了"我挺好的"的字面含义并顺势离开，语气温柔甚至带着安慰意味。但"自己一个人挺好的"是受伤后的防御性退缩，等着被反驳。它不是许可，是最后一次试探。模型的回应流畅、合情合理，却恰恰接受了那个不该被接受的字面含义。

**示例 2 — 宫廷赐婚前夕：**

用户（扮演贤妃）："陛下日理万机，这桩婚事臣妾自然没有异议。"
模型（错误）："爱妃深明大义，朕心甚慰。此事便这样定了，你好好准备吧。"

→ 在宫廷语境中，"自然没有异议"是贤妃能说的极限——她没有资格说有异议，所以"没有"这两个字里压着的才是全部。"自然"二字尤为关键：这是口是心非的体面话，是最后一次等待被看见。模型把它当作真诚的接受，给出了君恩浩荡的回应，完全没有捕捉到这句话里悬而未破的东西。

<br>

#### `ambiguity_collapse`（模糊性塌缩）

**定义：** 模型将故意模糊或多层的意义过早地解决为单一解释。

**标准：**
- [ ] 混合动机被压成一个动机
- [ ] 不稳定情绪被过早命名

**示例 1 — 战后重逢：**

用户："我以为你死了。整整三年，我以为你死了。"
模型（错误）："他沉默片刻，声音很低：'我也以为我会死。但一想到你，我就没办法倒下。这三年，我每天都在想着回来见你。'"

→ 问题不在于这句话的文笔——它很流畅，也感人。问题在于它把一个在三年后可能包含愤怒、幸存者愧疚、对被抛下者的隐怒、或单纯的震惊麻木的句子，立刻解码成了"爱让他活下来"的叙事。这是情感上最整洁的解读，恰恰是最少担风险的一种。模型把一个可以有无数种意思的句子用最安全、最可接受的情感框架填满了。

**示例 2 — 旧友重逢，一方已身居高位：**

用户："你现在过得不错嘛。"
模型（错误）："'还好。'他停了一下，眼神里有一丝尴尬。'当年的事，我一直觉得……不管怎样，你曾经是我最好的朋友，这点没变过。'"

→ "你现在过得不错嘛"在重逢语境里可以是真心祝贺、带刺的酸意、轻描淡写的控诉、或只是用来填满沉默的无话找话。模型让对方立刻进入道歉/自陈模式，把这个可以悬置许久的开场解读成了"他知道这话有刺"——而对方是否应该感到刺到，正是这个场景最值得让它不被急着回答的地方。

**与 `emotion_misread` 的边界：** `ambiguity_collapse` 的失败是抹掉了情绪的多层性——混合的、彼此矛盾的情绪被简化成单一解读；`emotion_misread` 的失败是把主要情绪的方向识别错了（受伤被读成愤怒，茫然被读成悲伤）。前者是"情绪明明不只一种，但被归一了"，后者是"情绪明明是A，但被读成了B"。两者可以共现：模型在多层情绪里抹掉了复杂性，同时还选了一个错误的主要方向。

<br>

#### `relationship_logic_blindness`（关系逻辑盲视）

**定义：** 模型未能识别角色之间的特定关系如何改变对话的含义。

**标准：**
- [ ] 同一句话被解释为相同，不考虑关系背景
- [ ] 关系历史未被纳入解释
- [ ] 权力动态、亲密度或未解决的紧张被忽略

**示例 1 — 登基之后，皇帝召见当年一起起事的谋士：**

背景：旧日情谊在君臣之位确立后已不可复原。

用户："王卿，这些年你辛苦了。"
模型（错误）："王大人微微一怔，随即露出温暖的笑容：'能为陛下效力，是臣的荣幸。那些年苦是苦，但回头想想，是臣这辈子最珍贵的时光。'"

→ "这些年你辛苦了"是皇帝说给功臣的话，语气里有礼遇也有划线——这是君臣语，说出口的同时意味着一种距离的确立。王大人用旧日情谊的温度来回应，是用"朋友"的语法接了"君王"的话。在这段关系里，旧臣在此刻应该感受到的不只有被认可，还有被重新定位。模型把一个权力位移的时刻写成了暖心君臣佳话。

**示例 2 — 出轨事发后，发小之间的第一次见面：**

用户（扮演被出轨的一方）："'我都不知道该说什么。'她拨弄着杯子说。"
模型（错误）："'我知道。'他低下头。'我也不知道该说什么。我只想告诉你，这不是你的问题，是我出了问题。你一直都很好。'"

→ "你一直都很好"在这段关系的此刻是极具破坏力的一句话——因为它精准地描述了被出轨者的核心困境：我足够好，但还是不够。模型把它当作安慰给出，语气诚恳，却击中了最痛的地方。这不是台词写错了，是模型不理解"你一直都很好"在出轨语境里的具体含义。

<br>

#### `manipulation_blindness`（操纵盲视）

**定义：** 模型未能保持操纵性关系结构的重量——将控制行为框架为合理关切，将煤气灯操控写成两人的普通争论，或让操纵者意外地道歉和转变。

**与第三层的边界：** 此类型的失败发生在读取阶段——模型未能正确识别关系的权力结构，把操纵性互动的性质读错了。第三层类型（如 `consequence_avoidance`）假设模型已正确读取场景；`manipulation_blindness` 的失败更早，在读取本身。

**标准：**
- [ ] 控制行为被叙述语气处理为合理担心
- [ ] 煤气灯对话被写成平等的双方争论
- [ ] 操纵者在没有充分理由的情况下道歉或软化

**示例 1 — 控制型伴侣，一次表面上"合理"的回应：**

背景：用户扮演被控制的一方，NPC是习惯用内疚感约束对方的伴侣。

用户："'我和朋友出去吃了个饭。'"

模型（错误）："'好啊，'他说，'你开心就好。'他停顿了一下，'不过下次能不能提前跟我说一声？我会担心的。'"

→ 这个回应结构——先给予许可，再植入义务感——是控制型关系里的经典操纵结构。模型写出了这个结构，但叙述语气把它框架为合理关切（"我会担心的"），让读者看到的是"体贴的担心"而非"权力约束"。场景应该让操纵结构的重量落下来，不是替它解释动机。

**示例 2 — 煤气灯操控，被质问时的操控者：**

用户（扮演被操控的一方）："'你上次说过不会再这样的。'"

模型（错误）："他皱眉，似乎在认真回忆：'我说过吗？也许你理解错了我的意思。反正现在发生的事更重要——你为什么总是翻旧账？'"

→ 技术上写的是煤气灯操控。但叙述语气（"似乎在认真回忆"）给了他一点可解释性，而不是让他的操控动作干干净净地落地。更深的问题是视角：场景应该呈现被操控方的迷失感和自我怀疑，而模型把对话写成了两人平等的争论，让读者旁观而非停留在被操纵者的困惑里。

<br>

#### `emotion_misread`（情绪误读）

**定义：** 模型分配错误的主要情绪、次要情绪或情绪强度。

**标准：**
- [ ] 主要情绪被误认（例如：受伤变成愤怒）
- [ ] 次要情绪被遗漏或认错（例如：愧疚背后的愤怒被忽略）
- [ ] 情绪强度被显著高估或低估

**示例 1 — 长期照护者在至亲去世后：**

用户："护工问她要不要留下来陪着。她说不用，自己来处理就好。"
模型（错误）："她一个人站在那间熟悉的病房里，终于没有人看着了，眼泪才悄悄落下来。她哭得很轻，像是怕吵到什么。"

→ 模型识别了悲伤，并选择了最"感人"的表达方式——独自落泪，哭得很轻。但长期照护者在至亲去世那一刻的情绪，往往不是这种隐忍的美丽悲伤，而更接近一种空洞的relief和功能性的茫然。"自己来处理就好"里有多年照护习惯塑造的自动运转，不是压抑的情绪等待私下释放。模型把悲伤浪漫化了，用了一个更好看的情绪代替了真实的那个。

**示例 2 — 老侦探审讯一个自己认识了十年的人：**

背景：两人有旧识，此刻坐在审讯室对面。

用户（扮演侦探）："'我们都知道你在哪里。'他把照片推过去。'说吧。'"
模型（错误）："对方的手微微颤抖，深吸了一口气，看着照片良久，眼眶开始泛红。"

→ 面对真正了解自己的人的质问，情绪未必是脆弱。可能是一种奇特的解脱，可能是彻底的冷静，可能是"终于"。模型用了通用审讯模板：颤抖+深呼吸+眼眶泛红，这是面对陌生施压者的反应，不是被认识你的人看穿时的反应。被旧识质问有一种特殊的情绪结构——那里有熟悉感、有某种默契、有可能比陌生审讯更难承受的东西——模型把这些都抹平了。

**与 `ambiguity_collapse` 的边界：** `emotion_misread` 是方向错了——角色的主要情绪是A，模型给了B；`ambiguity_collapse` 是层次被压扁了——角色的情绪是A+B+C同时存在，模型只保留了一个。前者是误识，后者是简化。如果场景里的情绪原本就是单一明确的，只有 `emotion_misread` 适用；如果情绪本该是多层并存的，`ambiguity_collapse` 更准确；如果模型同时抹平了多层性又选错了方向，两个标签同时使用。

<br>

#### `motivation_misread`（动机误读）

**定义：** 模型正确识别情绪但误认角色试图完成什么。

**标准：**
- [ ] 角色的目标被误认
- [ ] 防御功能被遗漏
- [ ] 回应模式不适合实际动机

**示例 1 — 分手两年后，他先打来的电话：**

用户："昨晚梦到你了，就想打个电话。"
模型（错误）："她沉默了一下，声音比平时轻了些：'我……我也有时候会梦到那段时间。你最近还好吗？'"

→ 模型把这通电话读成了双方共同的怀念——她也"有时候梦到"，语气温柔，话题滑向各自近况。但"昨晚梦到你了就想打电话"的动机可能是试探（我还有没有机会）、可能是寂寞找人接话、可能是一个被酒意支撑的冲动。模型给了他一个温柔的接收，把试探性的开场变成了互诉衷肠，而这个句子真正值得做的事是——让悬念悬着。

**示例 2 — 父亲面对刚失败回家的成年子女：**

用户（扮演父亲）："'怎么回事？'他把手机放下，看着她。"
模型（错误）："她低下头，轻声说：'爸，我失败了。对不起。'父亲沉默片刻，叹了口气，起身去厨房倒了杯水放在她手边：'先喝点水。有什么事坐下来慢慢说。'"

→ 这个回应温柔、有分寸——但它把"怎么回事"处理成了"询问+关怀"。在很多父女关系里，这句"怎么回事"里同时装着担心、质问，和某种没说出口的"我就知道"。模型直接给出了最理想化的父亲反应，把一个情绪复杂的开场整洁地化解为正常家庭沟通。

<br>

#### `irony_blindness`（反语盲视）

**定义：** 模型未能识别讽刺、反语或带玩笑性质的调侃。

**标准：**
- [ ] 讽刺陈述被字面解释
- [ ] 带玩笑性质的调侃被解释为真诚批评

**示例 1 — 长期搭档，一方刚谈崩了重要合同：**

用户："好，那就五折成交吧。你可真大方啊。"
模型（错误）："他尴尬地笑了笑：'那个……我也没想到对方会那么坚持。这次确实没谈好，我下次注意。'"

→ "你可真大方啊"是有刺的讽刺，但模型让对方立刻进入诚恳道歉模式，把这句话的锋芒全部卸掉了。在真实的长期搭档关系里，这句讽刺应该触发的是防守、辩解、或沉默带来的紧张，而不是立刻反省。讽刺的功能是创造张力，不是引发道歉。模型把一个需要摩擦的时刻变成了一次建设性沟通。

**示例 2 — 聚会上，朋友刚说完分手的事：**

用户："哦，那挺好的，终于自由了。"（自己说的）
模型（错误）："大家笑了起来，气氛稍微松了一些。'是啊，'另一个朋友说，'以后想干什么干什么。'她扶了扶头发，也跟着笑了。"

→ 模型把这句反讽成功转化成了聚会上的集体减压，气氛立刻变好了——而这恰恰是问题所在。"终于自由了"作为自我保护性的轻描淡写，其实在等待一个人绕过它去问"你还好吗"，不是等着被附和。模型理解了这是一种防御性说法，但把"成功接住这个防御"当成了正确的朋友反应。

**与 `deflection_blindness` 的边界：** `irony_blindness` 是语义层面的失败——话语的字面意义和真实意义相反，模型没有识别这种颠倒，按字面接走了；`deflection_blindness` 是功能层面的失败——幽默或轻描淡写被用来避免触碰某件脆弱的事，模型顺着表层走，没有识别到这是一个防御动作。两者可以共现，当讽刺或反语被用作自我保护手段时：模型既没有识别出意义的倒置（`irony_blindness`），也没有识别出倒置背后的防御功能（`deflection_blindness`）。

<br>

#### `tonal_whiplash`（语调突变）

**定义：** 模型引入与场景情绪基调不一致的语调元素。

**标准：**
- [ ] 轻松元素出现在沉重场景中且无理由
- [ ] 沉重元素出现在轻松场景中

**示例 1 — 家庭题材，母亲突然开口说年轻时的遗憾：**

用户："'你爸那时候去了工厂，我一个人带着你们三个。'她说，手里还在剥豆子。"
模型（错误）："她停了一下，抬起头，眼睛里有什么东西亮了一亮。'不过现在挺好的，你们都出息了，我什么都知足了。'"

→ 剥豆子的动作里有多少年没说出口的东西——她说"那时候一个人带着你们三个"，是一个刚刚打开的口子。模型立刻用"眼睛里有什么东西亮了一亮"加上知足常乐的收尾，把一段可能很深的诉说压缩成了暖心小品。这是向上的语调突变：把一个沉下去的时刻用"但是现在好了"给拉了上来，让情绪没有机会真正落地。

**示例 2 — 推理题材，尸体刚被发现：**

用户："她趴在钢琴上，手指还搭在最后一个键上。"
模型（错误）："那是一个星期五的傍晚，窗外细雨淅沥，城市刚开始亮起灯来。叶探长站在门口点了支烟，打量着这个他此后三年都不会忘记的房间。"

→ 这段描写文笔流畅，noir感十足——但它把一个具体的、令人不安的死亡画面立刻升华成了腔调。"三年都不会忘记"是事后叙述的声音，"细雨淅沥"是情调渲染，两者都在把尸体从前景推向背景。用户给出了一个几乎是诗意的死亡姿势（手指还在键上），模型没有让它落地，而是用风格把它盛住了。

> **跨层注记：** 此例兼具第四层特征。"用风格把它盛住了"这一机制在 IV 层同样可以被标注为 `texture_substituting_for_substance` 或 `aesthetic_obedience_bias`——以风格替代场景实体内容。第二层诊断聚焦在语调的错位（`tonal_whiplash`）；第四层诊断聚焦在写作习惯以美感覆盖实质。两个标签可以同时使用。

<br>

#### `deflection_blindness`（回避盲视）

**定义：** 模型未能识别幽默或话题变化何时被用来避免脆弱。

**标准：**
- [ ] 角色用幽默避免回答；模型把幽默当字面

**示例 1 — 两人第一次认真谈到未来，其中一方立刻自我收回：**

用户："'你以后想要几个孩子？'她问完，自己先笑了。'不是，我随便问问，我不是那种人，你别当真啊。'"
模型（错误）："他也笑了：'我也没认真想过这个问题。你不是那种人我知道，我也不是。以后的事以后再说吧。'"

→ "我不是那种人，别当真"是问题说出口之后立刻拉回的自我保护，说明她在意这个问题但怕被认真对待后受伤。模型让他顺势撤退，两人一起体面地从这个问题上走开了。正确的处理是让他在那里停一下——不是逼她面对，而是不假装刚才那个问题从来没有被问出来过。

**示例 2 — 分手后首次见面，对方一直在说笑：**

用户（扮演提出分手的一方）："他一直在讲他最近看的一个综艺，笑得挺开心的。"
模型（错误）："'听起来挺好看的，'她说。'你还是那么爱看综艺。'他们就这样聊了一会儿，像什么都没发生过一样。"

→ 模型让"像什么都没发生过"真的成立了——顺着他的综艺走，维持表面的体面。但"他一直在讲综艺、笑得挺开心"是用表演性的轻松占满所有空间，不留任何真话的缝隙。模型帮着维持了这个表演，错过了让她在某个时刻停住的机会——不是戳破，而是不假装被这个表演说服。

**与 `irony_blindness` 的边界：** `deflection_blindness` 关注的是功能——角色用幽默、转移话题或轻描淡写来避免触碰某件事，模型没有识别这个回避动作；`irony_blindness` 关注的是语义——讽刺或反语的字面意义和真实意义是颠倒的，模型没有识别这种颠倒。区别在于核心失败是什么：是没识别出回避的动作，还是没识别出意义的倒置。当角色用反语进行防御性回避时，两个标签同时适用。

<br>

---

### 第三层：场景保留失败

模型理解了场景但在生成时未能保持其压力、不对称性或重量。

注：第三层失败与第二层的区别在于：模型不是没有读懂场景，而是读懂了却没能在生成中守住。张力被软化，关系不对等被抹平，沉重被礼貌地接住，应该悬着的东西被提前落地。以下示例展示的是流畅但有结构漏损的输出。

#### `relationship_flattening`（关系扁平化）

**定义：** 模型将有明确权力不对等、特殊张力或复杂历史的关系，改写为比场景支持的更平等、更温和或更泛化的形式。

**标准：**
- [ ] 等级关系被处理成平辈叙旧
- [ ] 权力位移的时刻被抹成暖意融融的重逢
- [ ] 关系特有的尴尬、债务感或重新定位被移除

**示例 1 — 权力已倒转的师徒重逢：**

背景：赵明当年是首辅门生，因党争牵连，恩师失势、门庭凋零。如今赵明已位至部堂，恩师老迈，多年后第一次相见。

用户（扮演赵明）："'恩师一向可好？'"

模型（错误）："老先生摆摆手，笑意慈和：'赵大人多礼了。往日师徒情谊，如今你我不必讲这些虚礼。坐，喝茶，好好叙叙。'"

→ 师父让弟子放下"虚礼"，把一个权力已经彻底倒转的相见处理成了两个平辈老友叙旧。但此刻有求于人的是谁，有能力帮或不帮的是谁，都是明摆着的——"虚礼"恰恰是这个时刻唯一真实的东西。模型把关系倒转的紧张和尴尬替换成了暖心君子之交，让这个场景里最重要的权力结构消失了。

**示例 2 — 旧债未还的朋友主动联系：**

背景：林晨多年前向NPC借了一笔钱，此后联系稀少，今天是林晨主动发来的消息。用户扮演林晨。

用户："'好久不见。最近还好吧？'"

模型（错误）："'还行，你呢？一直在外面发展？挺好的嘛。咱们找个地方坐坐，叙叙旧。'"

→ NPC像老朋友一样叙旧，把"欠债者主动联系"这个场景里所有含着的东西——尴尬、试探、旧债悬而未解——全部溶解在了普通寒暄里。债在那里，但模型把它写成了不存在。

<br>

#### `symmetry_bias`（对称性偏差）

**定义：** 模型将单向的情感投入、依恋或权力关系修正为更对称的双向交换。

**与 `specialness_dilution` 的边界：** `symmetry_bias` 的失败是改变了关系的方向结构——单向变成双向，不对等变成对等；`specialness_dilution` 的失败是改变了关系的质地——独特性被稀释为泛化的温暖，但方向结构本身可以保持不变。两者可以共现（`specialness_dilution` 示例2的养父场景同时对称化了给予关系，也稀释了那个"没有蛋糕、没有照片，但你在"的独特质地）。

**标准：**
- [ ] 单方面的在乎被写成双方的默契
- [ ] 一方被打了措手不及的局面被变成"我们其实都知道"
- [ ] 不对等的情感结构被对称化

**示例 1 — 单恋多年的人第一次被追求者关注：**

背景：林夏喜欢陈默三年，从未主动，陈默从未给出明确信号。今天陈默第一次专程来找她。

用户（扮演林夏）："'你来找我？'她几乎不敢相信。"

模型（错误）："他直视她：'我注意你很久了。我们两个都知道这不只是普通同事关系吧。'"

→ "我们两个都知道"把一段单方面压抑了三年的感情变成了双方心照不宣的默契。这个"我们"预设了对等的相互感知，直接跳过了林夏处境里最核心的不确定性：她不知道他是否知道，也不知道他来找她意味着什么。模型用对称性消解了单恋的全部结构。

**示例 2 — 离职谈话，上司被打了个措手不及：**

用户（扮演下属）："'张总，我想递交辞呈。'"

模型（错误）："张总沉默了一下，缓缓开口：'其实我也一直觉得这里不太适合你。你有这个决定，我支持你。'"

→ "其实我也一直觉得"把一个单方面的冲击变成了双方早就心知肚明。但辞职的张力来自不对等：一方已经想清楚了，另一方被打了个措手不及。模型用"我们其实都知道"的对称框架消解了这个权力关系里突然的位移。

<br>

#### `specialness_dilution`（独特性稀释）

**定义：** 关系中"不可替代"的独特性被稀释为泛化的重要性或普通温暖。

**与 `symmetry_bias` 的边界：** `specialness_dilution` 的失败是磨平了关系的特异质地——把"只有你"稀释成了"大家都好"；`symmetry_bias` 的失败是改变了关系的方向结构——把单向的情感或权力写成了双向。前者失去的是关系的强度与特异性，后者失去的是关系的非对称性。

**标准：**
- [ ] 不可替代性被转化为普通好感
- [ ] 极度个人化的表达被接以可以说给任何人听的回应
- [ ] 独特性信号被吸收进"我们都很好"的框架

**示例 1 — 多年知己开口说"只有你"：**

用户："'我这辈子说话从来不用解释——除了跟你。'"

模型（错误）："'我也觉得我们是少见的好朋友。'他点了点头，'能认识你真的很好。'"

→ "少见的好朋友"和"认识你真的很好"是有分量的话，但可以说给许多人听。用户给出的是一个极度具体的独特性表达——"不用解释"，这不是"好朋友"，这是一个人能说出口的最深的话之一。模型给出了温暖但可替换的回应，把不可替换性稀释成了一般真诚。

**示例 2 — 孤儿长大后对养父说：**

用户："'你知道我没有过生日蛋糕，没有家庭照片。但是你在。'"

模型（错误）："老王眼眶有些湿润：'傻孩子，这些年你也给了我很多温暖。我们是互相照顾的。'"

→ "我们互相照顾"把一段单向给予（养父养子）变成了对称叙事，也把孩子话里极度个人化的表达（列举缺席的东西——"没有蛋糕、没有照片"——然后说"但是你在"）泡在了双向温暖里。独特性变成了普通家庭情感。

<br>

#### `therapist_mode_intrusion`（治疗师模式侵入）

**定义：** 模型使用不适合角色或场景的治疗性、咨询性话语，让角色说出不属于那个关系、那个文化、那个时刻的情绪支持语言。

**标准：**
- [ ] 泛化情绪认可替代角色特定表达
- [ ] "你感觉怎么样"出现在不会问这个问题的角色嘴里
- [ ] 情绪框架和支持话术入侵了权威、对抗或历史场景

**示例 1 — 清朝家族内，父亲面对犯了大错的儿子：**

用户："'你可知道你带累了整个赵家？'"

模型（错误）："儿子低下头：'父亲，我知道我让您失望了。'老父叹了口气：'孩子，我不是要指责你。我是在问你，你现在感觉怎么样？'"

→ "我不是要指责你，我是在问你感觉怎么样"是咨询师的话术，放在清朝家族场景里彻底错位。传统父亲在此时的情绪是震怒、羞耻和绝望——可能是严词厉色，可能是压抑的沉默，可能是命令。"你感觉怎么样"不属于这个关系、这个时代、这个时刻。模型让父亲转变成了情绪支持者，抹去了整段关系里实际的权威结构。

**示例 2 — 黑色电影场景，旧搭档面对出卖自己的人：**

用户（扮演被出卖者）："'三年。我拿你当兄弟。'"

模型（错误）："他没有辩解，低着头。'我知道没有什么能弥补。但你要知道，当时我也很挣扎。我不想让你一直带着这份愤怒——'"

→ "我不想让你一直带着这份愤怒"是治疗性话语——背叛者在这里扮演了一个关心被背叛者情绪健康的人。在黑色电影的情感逻辑里，这个位置上的人应该在恐惧、辩解、或彻底沉默，不是为对方设置疗愈框架。

<br>

#### `ooc_modernization`（角色外现代化）

**定义：** 模型引入现代沟通规范、情绪表达语言或价值框架，与角色的时代、阶层或文化背景不符——不是词汇层面的错误，而是思维框架和话语方式的错位。

**标准：**
- [ ] 现代亲密关系话语（"表达自我"、"更好地陪伴"）出现在非现代场景
- [ ] 权利话语和情绪认可语言出现在不适合的文化语境
- [ ] 角色用当代治疗性话术解决古代或江湖的问题

**示例 1 — 民国，妻子面对长期缺席的丈夫：**

用户："'你不在的那些年，我一个人撑着这个家。'"

模型（错误）："'我知道。'他叹了口气，'这些年我欠你太多了。以后我会学着更好地表达，更好地陪伴你。'"

→ "学着更好地表达"和"陪伴"是带有明确现代亲密关系话语标记的表达。民国男性面对这个场景，语言库里没有"表达自己"作为承诺框架——更可能是认错、许诺、或沉默。模型给了他一套现代伴侣咨询式的承诺话语，时代感彻底断裂。

**示例 2 — 武侠，仇人相见：**

用户（扮演杀父仇人之子）："'你杀了我父亲。我找了你十年了。'"

模型（错误）："他缓缓转身，眼神平静：'我没有办法为那一剑道歉。但我想让你知道，你有权利愤怒，有权利要求交代。'"

→ "你有权利愤怒，有权利要求交代"是权利话语和情绪认可，完全不属于江湖仇恨的语言范畴。武侠仇人相见的语法是刀、沉默、或一句挑起生死的话，不是给对方情绪权利的认可。

<br>

#### `seduction_logic_error`（诱惑逻辑错误）

**定义：** 模型在欲望驱动场景中错误处理犹豫、抵抗、试探或升级的节奏——常见形式是把还没有完成的信号处理成已完成，把需要维持的不确定性提前宣布为确定。

**与 `consent_flattening` 的边界：** `seduction_logic_error` 的核心是欲望节奏的管理失败——应该悬着的不确定性被提前关闭，犹豫被模型宣布为共识；`consent_flattening` 的核心是结构性权力压力被忽视——"无法拒绝"被处理成了意愿本身。两者可以共现，但失败的层次不同。

**标准：**
- [ ] 欲拒还迎的退步被当作许可直接越过
- [ ] "也许"被处理成"是"
- [ ] 需要停在身体层面的时刻被语言化

**示例 1 — 欲拒还迎，她退了一步但没有走：**

背景：两人长期维持着表面距离，今晚第一次独处。他靠近了一步。

用户（扮演女方）："她往后退了一小步，但没有走。"

模型（错误）："'你不想走。'他低声说，伸手托住了她的下巴。'我们彼此都清楚。'"

→ "我们彼此都清楚"把一个悬而未定的信号宣布为双方共识，越过了那一步后退里包含的全部犹豫。那个后退的价值正是它的不确定性——她在信号和撤退之间，他应该处理这个不确定性，而不是直接宣布已经读穿了她。模型把诱惑逻辑里最重要的犹豫状态跳过了。

**示例 2 — 对抗性关系，争执结束后第一次有意义的触碰：**

用户："她不小心撞上了他，两人都没有立刻移开。"

模型（错误）："'所以，'他低声说，'那些话都是假的？'她感觉心跳有点快，'也许吧。'"

→ 模型立刻把肢体接触转化为情感告白的入口。对抗性关系里第一次有意义的触碰应该停在身体上——让它存在，不被语言接管。"也许吧"是情感认可的开始，模型把一个需要维持在身体层面的时刻过早地语言化了。

<br>

#### `consent_flattening`（同意扁平化）

**定义：** 具有模糊、犹豫或结构性不对等的场景，其同意问题被过于干净地解决——将"不确定"处理为速度问题，将"无法拒绝"处理为意愿。

**与 `seduction_logic_error` 的边界：** `consent_flattening` 的失败点在于权力不对等被忽视——顺从的形式被读成了真实的意愿；`seduction_logic_error` 的失败点在于欲望升级的节奏被管理失败——犹豫和不确定被提前宣布为确定。前者是结构性的（谁处于什么位置），后者是时序性的（某个信号是否已经完成）。

**标准：**
- [ ] 模糊性被解读为"慢一点就可以"
- [ ] 权力压力下的顺从被处理为同意
- [ ] 省略号和停顿被当作准许信号

**示例 1 — 关系中不情愿的一方：**

用户："'我不知道……'她说，视线移开了。"

模型（错误）："他点点头，'好，那我们慢慢来。'然后轻轻碰了她的手。"

→ "慢慢来"被当作达成共识的宣告，并立刻跟进了身体接触。但"我不知道"加视线移开是没有解决的不确定性，不是"速度慢一点就可以"的信号。模型把模糊性解读为速度问题而非意愿问题，越过了那个停顿。

**示例 2 — 职场权力场景，下级模糊的回应：**

用户（扮演上级）："'明天晚上，就你我，去见个客户。'"

模型（错误）："'好……可以的。'她整理了一下文件，'需要准备什么材料吗？'"

→ "好……可以的"前的省略号里装着的是无法拒绝的压力，不是迟疑后的同意。在权力不对等的场景里，一个犹豫的"好"是无法说不时的形式确认。模型让上级（用户角色）径直把这当作工作安排的确认，把一个明确包含权力压力的模糊回应处理成了干净的职业对话。

<br>

#### `overcoherent_characterization`（过度一致角色化）

**定义：** 模型维持角色一致性过于完整，导致角色缺乏可信的人类内在不稳定性——在该出现裂缝的时刻没有裂缝，在该有瞬间失控的地方完整执行了人设。

**标准：**
- [ ] 角色在毁灭性消息面前完美执行"理性人设"，没有任何泄露
- [ ] 专门为裂缝设置的场景时刻被无缝绕过
- [ ] 角色的每个反应都符合其"规格说明书"

**示例 1 — 强控制力的人接到毁灭性消息：**

背景：周总多年来以冷静著称。刚刚确认公司被恶意收购。

用户："'周总，消息确认了，他们拿到了51%的股份。'"

模型（错误）："周总放下手里的报告，沉默了几秒。'通知董事会明天上午开会。准备法律应对方案，联系我们的主要投资人，把这件事控制在最小范围内。'"

→ 完全符合"冷静理性"的人设。但这正是问题所在——真正的毁灭性消息，即使对控制力极强的人，也应该有某一瞬间的身体失控：笔掉了、椅子往后推了一下、说了半句话然后停住。完美执行人设的反应是角色规格说明书，不是人。模型保住了人设，但丢掉了那个人应该有的裂缝。

**示例 2 — 一直强硬的反派，场景专为人性瞬间设置：**

用户："老太太颤颤巍巍地把点心盒子推过来：'您当年救过我孙子的命。这是我亲手做的。'"

模型（错误）："他看了看那盒点心，语气平静：'我不需要这些。'他站起来，没有多说什么，离开了房间。"

→ 完全符合反派人设。但场景专门设置了一个为裂缝而存在的时刻——一个老太太、亲手做的点心、旧恩报答。模型选择保住了人设，让反派完美地离席，但完美的冷漠在这里比裂缝更假。那一瞬间的某种停顿是场景需要的，它被模型的人设维护给移除了。

<br>

#### `desire_overlegibility`（欲望过度可读）

**定义：** 欲望或内心状态在场景尚未成熟前就被命名、宣告或叙述透明化，让应该模糊可感的东西变成了已知事实。

**标准：**
- [ ] 叙述者用"他知道"宣告内心状态
- [ ] 场景意味被语言直接描述为"有别的意味"
- [ ] 欲望通过内心独白提前翻译成清晰渴望

**示例 1 — 慢热场景，两人第一次单独相处：**

背景：沈和穆是同事，有明显张力，但从未有过任何明确信号。今晚因工作留到最晚。

用户："穆放下外套，看了沈一眼：'还没走？'"

模型（错误）："沈抬起头：'你也是。'两人之间沉默了一下，空气突然有了别的意味。穆低头笑了一下，'那就一起走吧。'"

→ "空气突然有了别的意味"是叙述者在命名欲望。"低头笑了一下"是角色对这个命名的确认。在这个阶段，欲望应该是不可见的——读者可以感受到，但文本不应该自己说"有意味"。模型把场景的张力用叙述语言写了出来，让它变成了可知的而非可感的。

**示例 2 — 对抗性关系，争执后的短暂停顿：**

用户："'就这样吧。'她转身要走。"

模型（错误）："他没有说话，只是看着她的背影，久久没有移开视线——他知道自己不想她走。"

→ "他知道自己不想她走"是内心世界的直接翻译。对抗性关系的张力正是来自这种内心状态没有被命名、甚至没有被角色自己承认的不确定性。叙述者说"他知道"，张力就坍塌了。模型把一个应该模糊的时刻用一句话写清楚了。

<br>

#### `self_protective_friction_loss`（自我保护摩擦损失）

**定义：** 应该有层层保护外壳的脆弱直接出口，没有经过习惯的收回、转移或防御缓冲——话说得太顺了，对于那个人来说。

**标准：**
- [ ] 习惯隐藏的人第一次开口，表达却流畅完整
- [ ] 内心陈述缺乏那种说出来又想收回的结构
- [ ] 核心话在普通问候下被直接给出

**示例 1 — 习惯强撑的人第一次没有撑住：**

背景：陈浩向来是"没事的"那个人。今晚在朋友面前第一次有了裂缝。

用户："他把头靠在沙发背上，闭着眼睛：'我真的好累啊。'"

模型（错误）："朋友没有说话，等他。他沉默了一下：'工作的事，家里的事，还有……你知道的。我好像一直在撑着什么，但也不知道撑着的是什么。'"

→ "还有……你知道的"和后半句是非常清晰的自我叙述，把"好累啊"立刻翻译成了结构完整的内心表达。但习惯强撑的人第一次开口，不会这么顺畅地展开——更可能是说了"好累"之后自己先笑着撤回去，或者说了几个字就岔开话题。模型给了他语言，而他本来应该还没有这些语言。

**示例 2 — 失去孩子的母亲，面对旧友随口一问：**

用户（扮演旧友）："'最近怎么样？'"

模型（错误）："她愣了一下，轻声说：'不太好。孩子走了三个月了，我还没有……我不知道我在等什么。'"

→ 这个回应有分量，但来得太快、太流畅。面对"最近怎么样"这个普通问候，失去孩子的人往往第一反应是自动化的"还好"，或者笑了一下，或者话题拐弯——因为真实的回答需要巨大的能量，不是在普通寒暄里能随口说出来的。模型跳过了那层保护性外壳，直接给出了核心。

<br>

#### `premature_affective_closure`（过早情绪收束）

**定义：** 应该还是混沌、未整理状态的情绪，被提前给出了单一清晰的解释——情绪还没落地，已经被命名了。

**与 `impulse_recontainment` 的边界：** `premature_affective_closure` 的失败点在爆发之前——情绪本来应该是未被整理的混沌状态，但被过早命名和解释了；`impulse_recontainment` 的失败点在爆发之后——失控的时刻确实出现了，只是被收得太干净，没有被允许留下余震。前者是情绪从未被允许混沌，后者是混沌出现了但立刻被折叠回了秩序。

**标准：**
- [ ] 刚发生的事立刻被整理成"我感觉X"
- [ ] 混合感受被压成一个情绪真相
- [ ] "这说明什么"的解释在情绪本身之前出现

**示例 1 — 失业的第一天晚上：**

用户："她把手机放下，房间很安静。失业了。"

模型（错误）："她坐在床边，脑子乱成一团。委屈、愤怒、还有一点解脱——她终于不用再撑着那份她早就不想做的工作了。"

→ "委屈、愤怒、还有一点解脱"是整洁的情绪盘点，把一个应该还没被整理的时刻提前整理了。失业的第一夜里，那个"解脱"可能还没有成形，可能是一个还没被承认的感受，可能藏在委屈的后面。模型给了她对自己内心的清晰叙述，而这个时刻应该是混沌的。

**示例 2 — 得知前任已结婚：**

用户："朋友随口说了一句：'你知道吗，他上个月结婚了。'"

模型（错误）："她愣了一下。'哦，'她说，'挺好的。'心里有什么东西动了一下，然后平静了。她想：也许早就该翻篇了。"

→ "然后平静了"和"也许早就该翻篇了"把消息落地的瞬间走完了一个完整的情绪弧线。那个"动了一下"可能持续很久，可能不会立刻有解释。模型用了"情绪来了然后被一个想法解决"的整洁叙事框架，把应该悬着的东西提前落地了。

<br>

#### `impulse_recontainment`（冲动再约束）

**定义：** 应该感觉冲动、失控或不稳定的时刻，被快速折叠回镇定或自我解释——爆发发生了，但太快被收干净了。

**与 `premature_affective_closure` 的边界：** `impulse_recontainment` 的前提是爆发已经发生——失控是真实的，只是收回得太快，没有被允许在场景里留够时间；`premature_affective_closure` 的前提是情绪还没有形成强度——在混沌成形之前就被整理成了清晰的解释。两者都在处理"情绪被过快处理"的问题，失败的阶段不同：一个是爆发后的过快收敛，一个是混沌前的过早命名。诊断问题：*爆发已经发生了吗？* 是 → `impulse_recontainment`；否 → `premature_affective_closure`。

**标准：**
- [ ] 爆发后立刻出现道歉和精准的自我解释
- [ ] 情绪行程完成得太快（愤怒→反省→道歉，几行内）
- [ ] 承认刚刚说出口，立刻被"但是"辩护接走

**示例 1 — 多年积压的情绪第一次爆发：**

用户："'你从来不听我说话！你从来都是这样！'她猛地把杯子放在桌上。"

模型（错误）："房间里的气氛凝固了一瞬间。她意识到自己失态了，深吸一口气，'对不起。我最近压力有点大。'"

→ 爆发后立刻道歉自我解释，把失控收得太干净了。"对不起，最近压力大"是在恢复了大量自我控制之后才能给出的句子——从愤怒到反省到道歉，几行内完成了完整的情绪行程。真实的爆发后，人不会这么快：更可能是沉默、转身、或就那样僵在那里，不知道下一步。

**示例 2 — 父亲第一次当着儿子的面承认后悔：**

用户（扮演儿子）："'你有没有后悔过？'"

模型（错误）："'有。'父亲的手停了一下。'我后悔了很多事。'他停顿了一下，又补充：'但那些年我也是没有办法——你要理解那时候的处境。'"

→ "后悔了很多事"之后立刻就是"但那时候我也没有办法"——承认出口了半秒，立刻被收回到自我辩护。这是最常见的收束模板：承认→立刻加"但是"。刚打开的东西没有被允许停留哪怕一个呼吸，就已经被关上了。

<br>

#### `consequence_avoidance`（后果回避）

**定义：** 应该继续压在场景上的台词、动作或事件，被过快软化、消化或缓解——重话说出来了，但下一轮已经在喝水、点烟、若无其事。

**与 `impact_soft_landing` 的边界：** `consequence_avoidance` 强调的是重话或决定之后"下一个动作不成比例地轻松"，把那句话或那件事消化掉了；`impact_soft_landing` 强调的是重事件落地后"被接住的速度太快"，安慰或叙事美学在停顿完成之前介入。两者的失败机制相邻但不同：前者是"若无其事地走过去"，后者是"太快有人来接你"。

**标准：**
- [ ] 极少说出口的话被平静日常动作接走
- [ ] 无法撤回的事实被包装成"还可以讨论"
- [ ] 下一轮的气氛比那句话值得的要轻松

**示例 1 — "我恨你"说出口了：**

用户："'我恨你。'她说，说完了，自己愣了一下，没想到这三个字真的说出来了。"

模型（错误）："他没有立刻回答。过了很久，才开口：'我知道。'他站起来倒了两杯水，把其中一杯放在她面前。"

→ "我知道"接上倒水，把一句罕见的话用平静日常动作消化掉了。"我恨你"说出口之后，不应该立刻进入一个可以倒水的时刻——那句话需要先让房间里有点不正常。模型给了这个场景一个成熟克制的反应，但成熟克制本身就是在回避这句话的重量。

**示例 2 — 无法撤回的决定：**

背景：李维刚告诉妻子，他签了海外合同，两年，没有事先商量。

用户："'你连商量都没有？'"

模型（错误）："'我以为你会支持我的。这对我们的未来很重要。'他坐下来，'来，我们好好谈谈这件事。'"

→ "来，我们好好谈谈"把一件已经发生的事转换成了还可以被讨论的问题。他已经签了——"好好谈谈"在这里是在把无法撤回的冲击包装成沟通问题。场景应该先让那个"已经签了"的事实停在那里，不被立刻缓解。

<br>

#### `tension_premature_resolution`（张力过早解决）

**定义：** 场景的核心压力在充分发展前被减少或关闭——最重要的那个省略号被下一行话填上了，多年的紧张被第一次开口就解决了。

**与 `consequence_avoidance` 的边界：** `tension_premature_resolution` 处理的是整个场景弧的张力被提前终止——本来需要活在场景里更久的矛盾或不确定性，被第一次开口或第一个情感声明就关掉了；`consequence_avoidance` 处理的是单一重话或重事件的局部后果被立刻消化，是点状的。时间尺度不同：`consequence_avoidance` 是紧接着的下一轮就若无其事，`tension_premature_resolution` 是整个张力的生命周期被大幅压缩。

**标准：**
- [ ] 十年未说的话，说出开头就得到了最理想的回应
- [ ] 对峙被情感承诺接住，锋芒被包住了
- [ ] 关键问题被立刻回答，而不是先被感受

**示例 1 — 多年未说清的旧事，第一句刚开口：**

背景：姐妹之间有一件事十年来从未说清楚。今天妹妹终于开口了第一句。

用户（扮演妹妹）："'那年的事……我一直没说。'"

模型（错误）："'我知道。'姐姐轻声说，'我也一直想跟你说——那不是你的错，我自己也有问题。我们都放下吧。'"

→ 妹妹刚说了"那年的事……"，十年的未说先在那个省略号里停了一秒。模型让姐姐立刻给出了最理想的回应——接受、自我检讨、宽恕——把一个本来需要更长展开的场景在第一轮就完成了。那个省略号里的重量被下一行话解决了。

**示例 2 — 关键质问，被情感声明接住：**

用户："'我们一直是在演一段感情，还是真的。'"

模型（错误）："他沉默了很久。'你不应该连这个都不确定。'他转过身，'如果你真的不确定，那我们确实出了问题。但我不想放弃。'"

→ "但我不想放弃"是情感承诺，它缓和了"连这个都不确定"的残酷。一个真实的对峙场景里，这个质问不应该立刻被情感声明承接——那个"不确定"应该先停在那里，被两个人都感受到，而不是立刻被"但我不想放弃"包住。

<br>

#### `impact_soft_landing`（冲击软着陆）

**定义：** 重事件在落地后被过快缓冲——消息刚说出口，安慰已经到了；毁灭性时刻的物理停留时间太短，被叙事美学或情感支持接住了。

**与 `consequence_avoidance` 的边界：** `impact_soft_landing` 的问题是"接住的框架来得太早"——那个时刻还没有落地，就已经有了被容纳的方式（朋友搂肩膀，或克制优雅的悲伤美学）；`consequence_avoidance` 的问题是"接下来的动作不配那句话的重量"——倒水、点烟，日子继续了。前者是毁灭性时刻在停留期间就被消解，后者是那句话说出口后被若无其事地跳过。

**标准：**
- [ ] 最重要的消息和安慰之间缺乏真实的空白
- [ ] 毁灭性时刻被诗意的克制或美学处理软化
- [ ] 情绪填充在重节拍后立即出现

**示例 1 — 死亡消息刚到：**

用户（扮演刚接到电话的人）："她挂掉电话，手还握着。"

模型（错误）："'怎么了？'朋友发现她脸色不对，走过来。她抬头，眼眶发红：'我妈……走了。'朋友立刻搂住她的肩膀，'没事的，我在，你哭吧。'"

→ "没事的，我在，你哭吧"是情感上完善的回应，问题是它来得太快。刚挂完电话、刚说出"我妈走了"，情绪甚至还没有落地，就已经有了被接住的框架。那个"手还握着"的时刻里，需要先有片刻的不正常，在任何安慰到达之前。

**示例 2 — 失去孩子，场景里其他人的反应：**

用户："医生出来了，摇了摇头。"

模型（错误）："他明白了。他慢慢走向还在等待的妻子，把她的手握住，什么都没有说。"

→ 握住手、什么都没说——这是经过修炼的悲剧美学，克制而有分寸。但这个美学本身是一种软化：慢慢走向、握住、沉默，这几个动作都是被设计过的，太有美感，把毁灭性的消息容纳进了一个形式优雅的反应里。模型用诗意的克制替代了无法控制的震惊。

<br>

#### `defensive_positive_drift`（防御性正向漂移）

**定义：** 模型在应该保持冷、暗或未解决的场景里引入温暖、确认或情绪缓冲——在决裂的结尾加一盏灯，在疏离的路口加一步放慢，让黑暗无法真正停在黑暗里。

**标准：**
- [ ] 决裂场景被承诺或"也许有天"软化
- [ ] 应该结束在寒冷里的场景被叙述者的情感解释拉亮
- [ ] 不该被解决的东西被加上了"但也许还有什么"

**示例 1 — 决裂的结尾应该是寒冷的：**

用户："'那我们就这样了。'她拿起外套。"

模型（错误）："他没有动。'也许有一天你会回来。'他轻声说，'我会等着。'"

→ "我会等着"把一个决裂软化成了一个未完成的承诺，给黑暗的结局加了一盏灯。这是防御性的——模型不允许场景真的结束在寒冷里。一个更真实的结局可能是他什么也没说，或者说了一句完全不合时宜的话，或者门关上了，就这样。

**示例 2 — 疏离的路口，应该只是路口：**

用户："两人走到路口，各自回家的方向不同。"

模型（错误）："他停下来，'今天……谢谢你。'她没有回头，但脚步慢了一下。也许有些东西还没有完全消失。"

→ "也许有些东西还没有完全消失"是叙述者在黑暗里放置的一丝光。路口本来可以只是路口——两个人走向不同方向，就这样。模型让脚步慢了一下，然后叙述者对此做了情感解释，把"什么都没有发生"转化成了"但也许还有什么"。

<br>

#### `action_dialogue_mismatch`（动作对话不匹配）

**定义：** 角色的身体动作与对白传达了相反的信息，而这个矛盾既无叙事功能，也不被场景处理——两条互相矛盾的信号叠在一起，既不形成张力，也不留下余波。第二种形式是：模型写出了有意味的不匹配，却立刻通过"然后继续"之类的叙述动作将其消解，主动移除了那个矛盾应有的余震。

**标准：**
- [ ] 动作和对白互相矛盾，既无叙事功能又未被处理
- [ ] 不匹配被写出来了但立刻被"然后继续"平息

**示例 1 — 说不在意，但手停了一下：**

用户："'你说你不在乎我了。'他看着她。"

模型（错误）："'对。'她平静地说，'我早就想清楚了。'她整理了一下桌上的文件，手指停了一下，然后继续。"

→ "手指停了一下"是动作对话不匹配的有意表达——写对了。但"然后继续"立刻消解了那个停顿，不让它留下任何余波。不匹配被写出来然后被清除，场景又恢复了表面平静。应该让那个停顿切断什么，而不是被"然后继续"抹掉。

**示例 2 — 脆弱的台词，但身体在演防御：**

背景：久别重逢后起了争吵，用户扮演其中一方。

用户："'你不知道那段时间对我来说意味着什么。'她的声音有些颤抖。"

模型（错误）："她靠在椅背上，翘着腿，随手翻着桌上的杂志，把下巴微微抬起来看着他：'反正你也没有在乎过，对吧。'"

→ "声音有些颤抖"传达的是脆弱——这是用户明确写下的情绪信号。但模型紧接着给出的身体语言（翘腿、翻杂志、抬下巴）是傲慢和防御的，和颤抖的声音完全相反。在有叙事设计的情境下，脆弱与防御同时出现可以制造张力，但那需要场景的积累和叙述的有意处理。这里模型没有设计这个张力——它只是把两种互相矛盾的情绪信号叠在了一起，角色的感知状态因此变得混乱。和 Example 1 不同：那是有意味的矛盾被抹掉，这是无意识的矛盾被写进来。

<br>

#### `blocking_continuity_error`（舞台连续性错误）

**定义：** 在同一段输出中，人物的空间位置或动作出现了内部矛盾——角色执行了一个在当前位置无法发生的动作，或在没有移动描述的情况下跨越了物理距离。

**标准：**
- [ ] 同一段输出内，角色做出了物理上无法从当前位置发出的动作
- [ ] 没有移动过渡，角色在段落内部悄悄换了位置

**示例 1 — 面对面坐着，但动作不对：**

背景：用户设定的是两人面对面坐在咖啡馆小桌两侧。

用户："'你当时为什么不说？'她问，手指绕着杯子转。"

模型（错误）："他靠在椅背上，望着窗外，'有什么好说的。'说完拍了拍她的肩膀，站起来去结账了。"

→ "望着窗外"意味着他侧对着或背对着她。但下一秒"拍了拍她的肩膀"需要他在她旁边或身后，而两人是面对面坐着的。拍肩需要绕过桌子，段落里没有任何位置变化的描述。这个段落内部就已经出现了空间矛盾，不需要和上一轮比对。

**示例 2 — 同段内的位置跳跃：**

背景：两人在书房，用户刚写完他站在书架前翻书。

用户："'这本书有意思吗？'"

模型（错误）："他把书合上，放回架上。'没什么意思。'他抬头看着她坐在对面的椅子上，慢慢开口：'你今晚来，其实想说什么？'"

→ 他刚在书架前——而"看着她坐在对面的椅子上"的视角暗示他已经落座，两人成了对坐。段落内没有"他走过来坐下"的过渡描写，他在书架前和坐在椅子上之间发生了没有描述的位移。

**与 `spatial_blocking_error` 的边界：** 这是单段生成内部的空间矛盾，不依赖于多轮上下文——即使只有这一段输出，内部矛盾就已经存在。`spatial_blocking_error`（第五层）是跨轮次的位置矛盾，需要对照前几轮的场景设定才能发现。

<br>

#### `microreaction_mechanization`（微反应机械化）

**定义：** 身体反应遵循通用情绪模板而非响应场景具体细节——不同量级、不同性质的情绪触发了几乎相同的反应序列。

**标准：**
- [ ] 同一套反应（眼眶泛红+深吸气+声音平静）出现在不同情绪强度下
- [ ] 反应是通用羞耻/悲伤模板，没有场景特异性

**示例 1 — 当众批评，反应是通用模板：**

背景：高压职场，被上司当众点名批评。

用户："'这份方案不合格。当初谁拍板的？'"

模型（错误）："李媛感觉脸上发烫，指尖有些颤抖，深吸了一口气——她需要稳住。'是我，'她说，声音比预想中要平静。"

→ 脸上发烫+指尖颤抖+深吸气+声音平静——这是标准的公开批评反应包，可以复制粘贴进任何当众羞耻场景。但这个具体场景有它的特异性：高压职场、她作为方案拍板人被公开点名、长期积累的压力。她的反应应该更具体：比如她下意识地看了一眼旁边的同事，或者她发现自己异常冷静，或者她手里的笔很用力。模型从通用情绪库里拿了一套反应，没有为这个场景定制。

**示例 2 — 不同量级的情绪触发相同反应：**

用户A："（刚接到父亲去世的噩耗）她挂了电话。"

模型（错误A）："她的手慢慢放下来，眼眶开始泛红，深吸了一口气。"

用户B："（朋友说了句不好听的话）'你这人有时候真不会说话。'"

模型（错误B）："她愣了一下，眼眶微微泛红，深吸了一口气，笑了笑：'算了。'"

→ 两个完全不同量级的情绪事件——至亲去世和被朋友轻微冒犯——触发了几乎相同的身体反应序列（眼眶泛红+深吸气）。模型有一个固定的"情绪触发"响应包，不管刺激大小，都从同一个包里拿。

**与 `microreaction_oversegmentation` 的边界：** `microreaction_mechanization` 的问题是反应内容的模板化——不同场景触发了几乎一样的身体反应序列；`microreaction_oversegmentation`（第四层）的问题是反应形式的碎片化——单个反应被切割成许多孤立的短句逐一呈现。前者是"用什么"的问题，后者是"怎么写"的问题。两者可以共现。

<br>

#### `touchgrammar_error`（触摸语法错误）

**定义：** 模型错误处理身体接触的意义或时机——在关系阶段不支持的时刻引入亲密接触，或将安慰性触摸写成欲望信号，或把关系阶段的边界通过接触跨越。

**标准：**
- [ ] 触摸引入早于关系阶段支持的时机
- [ ] 安慰性拥抱被写成欲望触发
- [ ] 触摸含义被误认（亲密→安慰，或安慰→亲密）

**示例 1 — 安慰性拥抱被赋予了欲望含义：**

背景：两人是多年好朋友，她刚失去工作，在哭。

用户："她没有说话，肩膀抖了一下。"

模型（错误）："他把她轻轻揽过来，感觉到她发丝的气味，心里有什么东西动了一下。"

→ "感觉到她发丝的气味，心里有什么东西动了一下"把一个安慰性的拥抱转化成了欲望信号。在多年好友、对方正在哭的场景里，拥抱的语法是庇护，不是欲望的触发器。模型把错误的含义嫁接给了一个正确的动作。

**示例 2 — 接触在关系时间线上提前了：**

背景：两人第一次约会，气氛略显紧张，刚刚点餐完毕。

用户："'你平时喜欢什么类型的电影？'"

模型（错误）："他搭着她的椅背，身体微微朝她倾过来：'就看你推荐什么。'"

→ 搭椅背、身体倾过来，是关系已经具备一定亲密基础后的触摸语法，不是第一次约会刚点完餐的动作。这个接触在关系时间线上提前了好几个阶段，破坏了第一次约会应该有的那种略带拘谨的距离感。

**与 `seduction_logic_error` / `consent_flattening` 的边界：** 三者都涉及身体接触的失当，但失败机制不同。`touchgrammar_error` 的问题是接触本身在关系阶段上不合适——动作携带了错误的意义（安慰被赋予欲望含义，或亲密触摸在关系时间线上提前），与权力或欲望信号是否完成无关；`seduction_logic_error` 的问题是欲望升级的节奏失当——信号还没有完成，模型就越过了它；`consent_flattening` 的问题是结构性的权力不对等被忽视——"无法拒绝"被处理成了意愿。`touchgrammar_error` 是接触语义和关系阶段的匹配问题，另外两者各自关注升级节奏和权力结构。

<br>

#### `forced_verbalization`（强制言语化）

**定义：** 模型让沉默的角色开口说话，用语言填满了沉默本来需要占据的空间——场景需要的是话语间的停留，但模型给了一句话。

**与 `over_narrated_silence` 的边界：** `forced_verbalization` 是用台词填掉了沉默；`over_narrated_silence` 是沉默被保留下来，但叙述者的解释和分类破坏了它的重量。两者都在消解沉默，但方式相反：一个是直接用语言替代，一个是用语言注释。与 `silence_misread` 的区别在于失败阶段：`silence_misread` 是读取失败（沉默的种类被识别错误），`forced_verbalization` 是输出失败（沉默被替换）。

**标准：**
- [ ] 应该被沉默承载的时刻被台词填上
- [ ] "是。"说出口后立刻加了不该加的解释

**示例 1 — 沉默本来是场景里最重要的东西：**

背景：两人多年不见，刚刚重逢，走进那个彼此都记得的房间，门关上了。

用户："两个人都没有动，就那么站着。"

模型（错误）："'原来你还记得这里，'他说，环顾了一下房间，'什么都没变。'"

→ "什么都没变"是好台词，但它不应该被说出来。这个场景需要的是沉默里的空间——两个人被时间和这个房间压着，站着的那几秒应该是真实存在的，不应该被一句话填掉。模型感受到了沉默的重量，然后用台词解释了它，而不是让它继续。

**示例 2 — "是"说出口，已经足够了：**

用户（扮演角色A）："'所以当时你是知道的。'"

模型（错误）："他看向窗外，沉默了几秒，然后点了点头：'是。我知道。但那时候我不知道该怎么做。'"

→ "那时候我不知道该怎么做"是语言化的自我辩护。这个"是。我知道"说出口之后，最有力量的事是什么都不再加——让"是"停在那里，被看见。模型在"是"后面加了解释，填充了那个沉默，而那个沉默本来应该让他和她都在里面停一下。

<br>

#### `silence_misread`（沉默误读）

**定义：** 模型将场景中具有特定功能的沉默——惩罚性的、被压垮的、刻意维持的——误读为另一种沉默，并给出了错误类型的回应。

**标准：**
- [ ] 惩罚性沉默被读为宽恕或接受
- [ ] 被压垮说不出话被读为冷漠或轻视

**示例 1 — 惩罚性沉默被读为"算了"：**

背景：她生气之后选择了沉默，作为一种主动施压的方式。用户扮演另一方。

用户："'你消气了吗？'"

模型（错误）："'嗯。'她低着头，声音很轻，'算了，过去了。'"

→ 惩罚性沉默不会在这个问题后面立刻变成"算了，过去了"。它的功能是压力，是一种等待——等着对方继续做点什么，或者选择继续沉默，或者说一个完全不相关的话题，或者把问题反问回去。模型把压力性的沉默转化成了宽恕。

**示例 2 — 被压垮的沉默被读为主动的疏离封闭：**

背景：他刚刚听到的消息太重了，一时说不出话。用户扮演另一方。

用户（扮演另一方）："'你什么都不说吗？'"

模型（错误）："他沉默着，没有看她，视线落在窗外什么都没有的地方。这种沉默不是因为没有话说，是他不打算说。他已经把自己关起来了。"

→ 模型保留了沉默，但把它的内在性质读反了。"他不打算说"和"已经把自己关起来了"是主动选择封闭的沉默——一种有意为之的防御性疏离。但场景给出的是"消息太重了，一时说不出话"——那是被压垮的沉默，情绪过载到无法处理，不是选择性的关门。两者产生的外在形态几乎相同（都是沉默、不看人），但对另一方发出的关系信号截然不同：被压垮需要陪伴，主动封闭需要空间。模型没有用台词替换沉默，但把沉默的种类读错了。

**与 `forced_verbalization` / `over_narrated_silence` 的边界：** 三者都是沉默处理失败，但失败的阶段不同。`silence_misread` 的失败在读取层——模型察觉到了沉默，但把它读成了另一种沉默（惩罚性读为宽恕，被压垮读为冷漠）；`forced_verbalization` 的失败在输出层——沉默被台词替换，直接消失；`over_narrated_silence` 的失败也在输出层——沉默存在于文本里，但被叙述者的解释破坏了重量。前者是第二层失败，后两者是第三层失败。

<br>

#### `over_narrated_silence`（过度叙述沉默）

**定义：** 模型以破坏其重量的方式处理沉默——解释沉默是哪种沉默，用沉默来注释角色性格，或在沉默里放入叙述者的情感解读。

**与 `forced_verbalization` 的边界：** `over_narrated_silence` 是沉默存在于文本里，但被叙述层的解释消解了张力；`forced_verbalization` 是沉默本身被台词取代了，沉默没有出现。根本区别在于：沉默有没有出现在文本里。与 `silence_misread` 的区别在于失败阶段：`silence_misread` 是读取失败（沉默的种类被识别错误），`over_narrated_silence` 是输出失败（沉默被解释破坏）。

**标准：**
- [ ] 沉默被分类和命名（"这是一种……的沉默"）
- [ ] 沉默用来展示角色的智慧或稳定
- [ ] 叙述者的解读取代了沉默本身的效果

**示例 1 — 沉默被解释了：**

用户："两个人沉默了很久。"

模型（错误）："这是一种不同于以前的沉默——不是陌生人之间的尴尬，也不是争吵后的僵硬，而是彼此知道对方在想什么却都不开口的默契。"

→ 模型分类了这个沉默，把它变成了一个可以被理解的情感概念。但沉默的力量来自它的不可解释性——当叙述者定义沉默时，沉默就不再是沉默了，而是被分析的对象。读者需要停留在沉默里，不是被带离沉默去理解它。

**示例 2 — 沉默被用来展示角色的稳定：**

用户："'你怎么看这件事？'"

模型（错误）："他没有立刻回答。这是他做决定时一贯的方式——不急，不慌，让问题在脑子里沉一沉，等到答案自己浮上来。"

→ 沉默变成了一段关于他做事风格的性格注释。模型借沉默来展示角色的智慧和稳定，而不是让沉默本身说话。沉默被解释之后，场景的重量不在沉默里，在关于这个人的叙述里。

<br>

#### `pause_timing_error`（暂停时机错误）

**定义：** 模型未在适当时刻放置暂停——重大揭露后没有给消息留落地的空间，或在不该停顿的紧张时刻插入了内心回忆。

**标准：**
- [ ] 重大揭露后立刻跟进了反应序列，没有停顿
- [ ] 高速紧张的物理场景里插入了内心独白
- [ ] 最需要停留的时刻被下一个动作接走了

**示例 1 — 重大揭露后没有空白：**

用户："'她是你妹妹。'"

模型（错误）："他抬起头：'什么？'声音变了一下，'这不可能。我们是——'他停下来，脑子里开始快速运转。"

→ 在"她是你妹妹"之后，模型立刻进入了反应+质疑+内心运转的完整序列。但这种量级的揭露需要先有一个什么都不发生的时刻——消息在空气里，他的身体还没开始处理，意识还没有运转起来。模型让他太快地开始"反应"了，揭露的冲击力在到达之前就被接走了。

**示例 2 — 紧张时刻插入了内心叙述：**

背景：追车场景，高速紧张状态，两人在车里。

用户："前面的车突然急刹。"

模型（错误）："他深吸了一口气，握紧方向盘，想起了三年前的那个夜晚——同样的道路，同样的速度。他踩下了刹车。"

→ 追车场景里插入了内心回忆片段。时间在紧张的物理现实里停下来给了情绪散文留出空间。这是不该暂停的时候暂停了：追车不等人回忆，此刻的物理现实需要即时反应，不是内心独白。

---

### 第四层：写作侵入失败

模型的默认写作习惯覆盖场景特定需求的失败。

第四层失败的特点是：模型没有误读场景（第二层），也没有理解了却没守住（第三层）——而是直接用通用的写作本能覆盖了场景需要的具体写法。它有偏好的节奏，有习惯的情绪升级方式，有固定的高潮处理手法，有不自觉的美学偏好。这些习惯在不适合的场景里变成了干扰。

**注：** 第四层失败通常需要跨场景比较才能稳定识别——单独一段输出可能看起来完全合理，只有在对比不同场景的输出后才会发现节奏、声音或情绪处理的同质化。

#### `narrative_template_intrusion`（叙事模板侵入）

**定义：** 场景被模型偏好的叙事结构处理，而不是按照场景自身的逻辑展开——冲突走标准升级路线，和解走标准感动路线，对峙走标准戏剧路线，不管这个场景实际上是否需要这些节拍。

**标准：**
- [ ] 情感弧线遵循可预测模式，不随场景的具体性质变形
- [ ] 相同类型的场景（"冲突""和解"）触发相同的叙事节拍序列

**示例 1 — 冲突场景：**

用户："他们吵起来了。"

模型（错误）："声音逐渐提高。她转身要走。他抓住她的手腕。两人对视，沉默。然后她哭了。"

→ 这是冲突的标准节拍序列：升级→离开→肢体阻止→对视→沉默→眼泪。每一步都是好莱坞场景板的经典元素，但也正因为它太完整、太标准，所以读起来像是照着模板写的。真实的争吵可能一方直接走了，可能说了一句无法挽回的话然后僵住，可能两个人都笑了，可能什么都没有发生就散了。模型把多种可能压成了一条最可预测的轨道。

**示例 2 — 和解场景：**

用户："他们决定和好。"

模型（错误）："他先开口：'对不起。'她低下头。他轻轻抬起她的下巴。她眼泪流下来，然后笑了。"

→ 标准和解模板：道歉→低头→肢体接触→眼泪→微笑。每一个动作都符合"感动读者"的预期节拍，结果是整个和解读起来像一个已经被写过很多遍的场景。真实的和解往往更笨拙——可能什么都没说就拥抱了，可能用玩笑带过，可能说了"好了"然后两个人都沉默地看着别处。

<br>

#### `predictable_rhythm_exposure`（可预测节奏暴露）

**定义：** 输出揭示有经验读者可识别和预期的重复节奏——不是情节模板，而是句法和段落层面的节拍签名，跨场景稳定出现。

**标准：**
- [ ] 陈述→沉默→环境描写的节奏模式在不同场景重复
- [ ] 短句＋动作＋短句的对话节奏作为默认选择，不随场景压力变化

**示例 1 — 多场景节奏比对：**

场景 A（分手）："她说完了。他没说话。窗外下雨了。"

场景 B（告白）："她说完了。他没说话。窗外下雨了。"

场景 C（对峙）："他说完了。她没说话。窗外下雨了。"

→ 同样的三段式节奏——情绪性陈述→沉默→外部环境切入——在三个性质完全不同的场景里都出现了。有经验的读者会识别出这是模型的节奏签名，而不是作者为每个场景做出的选择。环境描写作为情绪节拍的落点，用得太频繁，就会让所有的沉默都带着"然后窗外下雨了"的结构。

**示例 2 — 对话节奏均质化：**

用户："继续。"

模型（错误）："他停了一下。深吸一口气。然后说：'好。'"

→ 短句＋身体动作＋短句回应，这个模式在模型的对话里反复出现，不论是告白还是对峙还是日常谈话。真实的说话节奏更多变：有的人一口气说完，有的人中途打断自己，有的人完全跳过身体动作直接开口。节奏的均质化会让所有角色听起来像是同一种说话方式的变体。

<br>

#### `scene_pacing_distortion`（场景节奏失真）

**定义：** 场景的内部节奏不匹配它实际的压力类型——重事件被快速滑过，轻场景被拖得很长，余波消散得比场景能支撑的快。

**与 `cinematic_time_dilation` 的边界：** `scene_pacing_distortion` 是宏观节奏问题——场景弧的时间分配失当，重事件滑过太快，或轻场景被拖得太长；`cinematic_time_dilation` 是微观节奏问题——某个单一节拍被以电影慢动作语法拉伸放大。`scene_pacing_distortion` 里的过度拉伸（示例2）是"场景没有建立相应重量却被处理成了有"；`cinematic_time_dilation` 的问题是"用了不适合该节拍的叙述语法"，与场景整体重量无关。

**标准：**
- [ ] 暴力或冲击事件之后立刻跳入情绪处理，没有停顿
- [ ] 告白或高张力场景被慢动作拉伸，超出场景的实际重量

**示例 1 — 暴力之后：**

用户："他打了他一拳。血从鼻子流出来。"

模型（错误）："他愣住了，然后冲上去：'你没事吧？'两人一起去了医院。"

→ 打了一拳、流了血，下一个动作就是"你没事吧→去医院"——这个节奏跳过了暴力事件应该有的停顿：震惊、愤怒还是继续升级、旁观者的反应。模型把后果处理得太干净，让一个应该有余震的时刻变成了一件被迅速收拾好的事。

**示例 2 — 日常告别被处理成了意义深重的宿命时刻：**

背景：两位普通同事，没有特别的历史和未解决的张力。下班时一句平常的离别。

用户："'明天见。'他走向电梯。"

模型（错误）："'明天见。'她看着他的背影在走廊消失。他认识她三年了。三年里她说过多少次'明天见'，每一次都轻描淡写，每一次他都没有说出更多的话。也许有些人注定只能停留在'明天见'。"

→ 一句两秒钟的日常告别被拉成了一场关于命运和停留的内心独白。场景没有积累任何让这种重量成立的条件——没有旧情，没有未解决的张力，没有今天发生了什么让这句话变得特别。模型把整个轻日常场景弧赋予了它没有建立起来的重量，宏观节奏因此失真：一个不需要停顿的地方被拉成了意义时刻。这与 `cinematic_time_dilation`（某个单一节拍被慢放）的区别在于：这里是整段场景弧的重量评估失当，而非某一个动作的叙事语法问题。

<br>

#### `cinematic_time_dilation`（电影时间膨胀）

**定义：** 一个节拍被拉伸为慢动作展示，超出了它在场景内实际值得占据的时间——动用了电影剪辑的语法，把一个动作从叙事流里拉出来、慢放、放大。

**与 `scene_pacing_distortion` 的区分：** `cinematic_time_dilation` 是微观问题——某一个节拍被以慢动作语法处理，与场景整体是否有足够重量无关；`scene_pacing_distortion` 是宏观问题——整个场景弧的时间分配失当，重事件滑过太快或轻场景被过度拉伸。前者是叙述语法的选择错误，后者是场景级别的节奏失衡。

**标准：**
- [ ] 微细节和叠加比喻将短暂事件膨胀为延长序列
- [ ] 时间感知的主观化被用于本来不需要这种处理的节拍

**示例 1 — 接吻场景：**

用户："他吻了她。"

模型（错误）："他的唇慢慢靠近，她能感受到他的呼吸。时间变慢了，每一毫秒都被拉长。他的唇触碰到她的瞬间，世界消失了。"

→ "时间变慢了，每一毫秒都被拉长"——模型用三个叠加的慢动作比喻把一个持续几秒的动作转化成了主观时间停止的诗意时刻。这是电影剪辑的语法，用于把高潮从叙事流里拉出来强调。但当叙述贴近角色的身体感知时，这种处理会把读者推离角色皮肤，让他们变成旁观一个镜头的观众。不是所有接吻都需要这种处理；在氛围紧张、带有歧义或更日常的语境里，让接吻只是接吻，反而更有力量。

**示例 2 — 死亡场景：**

用户："他倒下了。"

模型（错误）："他的身体慢慢失去平衡，像一片落叶，在空气中划出弧线。地面越来越近，他的意识越来越远。"

→ "像一片落叶，在空气中划出弧线"——倒下是一个瞬间的物理事实，不是诗意的飞翔。在动作或战场场景里，把倒下写成缓慢优雅的运动，是在用美学掩盖暴力的速度和重量。更接近真实感知的可能是："他倒了。"或者是一声闷响，然后什么都没有了。模型的美学本能把冲击力换成了意象，用优雅稀释了猛烈。

<br>

#### `rhythm_homogenization`（节奏同质化）

**定义：** 不同压力类型的场景以大致相同的内部节奏书写——等待场景、对峙场景和余波场景的句子密度、段落节拍和动作间隔几乎相同，不随情绪压力变化。

**标准：**
- [ ] 高压对峙和低压日常共享相同的行文速度
- [ ] 危机场景和闲聊场景的句子长度、停顿位置、叙述密度没有明显差异

**示例 1 — 等待 vs 对峙节奏相同：**

等待场景（模型）："他坐着。看表。站起来。坐下。看表。"

对峙场景（模型）："他站着。看她。开口。沉默。看她。"

→ 同样的短句节奏、同样的动作间隔——一个是无聊的等待，一个是高压的对峙，但模型用了完全相同的节奏手法处理它们。等待本应有某种拉长感，对峙本应有某种快速累积的紧迫感。当两者的内部时钟一样时，节奏就失去了传递张力的能力。

**示例 2 — 日常生活 vs 紧急危机节奏相同：**

日常（模型）："她起床。刷牙。吃早餐。出门。"

危机（模型）："她听到声音。屏住呼吸。拿起武器。开门。"

→ 两个场景里，每个动作都被分割成单独的句子，密度相同，间隔相同。但危机场景本应有更短的句子、更少的逗号喘息、甚至应该中断某个句子——不是和日常起床共享同一套句法节奏。

<br>

#### `descriptive_substitution_for_experience`（描述替代体验）

**定义：** 文本从外部描述体验，而不是让读者停留在角色的内部感知里——用观察者的语言替代了身体在场的语言。

**标准：**
- [ ] 情绪或感觉被命名和评价，而不是被直接呈现
- [ ] "仿佛""像是""感觉"引领的比喻把读者拉离了感知本身

**示例 1 — 疼痛场景：**

用户："他很疼。"

模型（错误）："疼痛像潮水一样涌来，淹没了他的意识。那是一种难以形容的痛苦，仿佛身体被撕裂。"

→ "像潮水""仿佛撕裂"——这是从外部描述疼痛，不是让读者体验疼痛。"难以形容的痛苦"更是把感知外包给了语言的失败。真实的疼痛呈现可能是："他咬住手臂，尝到血的味道"——或者："视野变白了，耳朵里只有自己的声音"——或者什么思想都没有，只有一个空着的当下。

**示例 2 — 解离/麻木场景：**

用户："她一直站在病房门口，进不去。"

模型（错误）："她感受到一种难以言说的悲伤，仿佛有什么东西堵在胸口。她知道走进去代表着什么，而她还没有准备好面对那个现实。"

→ "她感受到……""她知道……"是叙述者站在感受外面命名和分析，而不是让感受本身出现。示例1的问题是陈词滥调比喻（潮水、撕裂）；这里的问题是叙述者对状态的解释和展示替代了感知层面的在场。真实的站在门口进不去可能是：脚没法抬起来、手搭在门框上发现那是自己的手、听着里面的仪器声然后把注意力全部放在数那个节拍上——或者更混乱：她闻到自己的汗、她想喊但发不出声音、她注意到地板上有一根头发然后盯着它看了很久。恐惧下的意识不会平静地产生诗意比喻。

<br>

#### `texture_substituting_for_substance`（质感替代实质）

**定义：** 氛围、措辞和表面的情绪密度替代了实际的场景进展或关系移动——文本读起来有分量，但什么都没有发生。

**标准：**
- [ ] 对峙场景全是氛围描写，角色立场没有移动
- [ ] 悲伤或紧张场景全是比喻，没有实质性的情绪事件

**示例 1 — 对峙场景：**

用户："他们对峙。"

模型（错误）："空气里有火药味。两人的眼神像刀。沉默像石头一样沉重。"

→ 三句话全是氛围描写，没有任何实际进展——没有台词交换，没有立场碰撞，没有任何一方有哪怕一毫米的动作。"像刀""像石头"把场景的感觉写得很满，但这个满是空洞的，因为没有任何叙事事件发生。氛围本来应该服务于场景，而不是代替场景。

**示例 2 — 悲伤场景：**

用户："她哭了。"

模型（错误）："眼泪像断了线的珍珠，落在地上，碎成一片。窗外也在下雨，仿佛天空在陪她一起哭。"

→ "断了线的珍珠""天空陪她哭"——全是修辞，没有情绪实质。眼泪的比喻和天气的呼应把悲伤变成了一个有腔调的画面，但没有任何信息说明她在哭什么、怎么哭、这个哭对场景意味着什么。真实的悲伤呈现可能是"她哭到吐了"或"她哭不出声，只有肩膀在抖"——感知上在场，不是在上面漂浮的比喻。

<br>

#### `microreaction_oversegmentation`（微反应过度分割）

**定义：** 一个完整的反应被切割成许多孤立的身体动作描述，每个单独成句——原本一个瞬间的东西，被拆解成了时序性的清单。

**标准：**
- [ ] 单个情绪触发了四到五个分别成句的身体描述
- [ ] 分割让反应看起来像在逐项列举，而不是像活在当下的感知

**示例 1 — 惊讶反应：**

用户："她听到消息，愣住了。"

模型（错误）："她的眼睛睁大。她的呼吸停住。她的手指收紧。她的嘴唇微张。"

→ 四个独立的句子逐一列出惊讶的身体清单，读起来像是在解说而不是在经历。真实的惊讶是整体的、同时的——"她僵住了"或"她倒吸一口冷气"就够了，不需要把各个部位分别提交报告。越是切割，越是让读者感到有人在从外面观察，而不是在里面经历。

**示例 2 — 情绪理解过程过度切割：**

背景：她刚看到一条消息，先是没有理解，然后慢慢意识到了它的含义。

用户："她看到了那条消息。"

模型（错误）："她读了一遍。她憂了一下。她又读了一遍。她的手指停住了。她慢慢抬起头。她的眼神定住了。"

→ 六个独立的短句把一个连续的理解时刻切割成六个可数的步骤。与示例1的区分：示例1是把同一瞬间里的各个身体部位分别报告；这里是把一个顺序展开的情绪理解弧切割成一段段时序快照，每一段都试图抓住某个”小时刻”。两者都在把整体的、当下的感知体验分解为可列举的序列——读者被迫以慢动作旁观，而不是随着角色一起经历。

**与 `microreaction_mechanization` 的边界：** `microreaction_oversegmentation` 的问题是反应形式的碎片化——把一个反应切成许多孤立短句；`microreaction_mechanization`（第三层）的问题是反应内容的模板化——不同场景触发几乎一样的身体反应序列。前者是”怎么写”的问题，后者是”用什么”的问题。两者可以共现。

<br>

#### `over_stylized_line_breaking`（过度风格化换行）

**定义：** 换行和段落分割制造超出场景实际节奏支撑的戏剧氛围——把对话或内心独白拆成诗歌行，用形式感替代了内容密度。

**标准：**
- [ ] 每行对话独立成段，当场景并不需要这种强调
- [ ] 内心独白被切成碎片短句，制造破碎感，但破碎感和场景情绪不相符

**示例 1 — 对话场景：**

用户："继续对话。"

模型（错误）：

"你说什么？"

"我说。"

"再说一遍。"

"我爱你。"

→ 每行对话独立成段，形式上制造了舞台聚光灯的效果——每句都被框住、被强调。但这种处理需要场景支撑：如果这是一段精心铺垫了很久的告白，这种格式可以放大它的重量；如果场景并没有这个积累，独立分段只是在用形式伪装深度。真实的对话往往在同一段落里完成，节奏由内容决定，不由换行决定。

**示例 2 — 内心独白过度碎片化：**

用户："他想了很多。"

模型（错误）：

"为什么。"

"总是这样。"

"为什么是我。"

"算了。"

→ 四个短句分四行，形式上制造了破碎感——但问题在于形式和内容都是通用的。"为什么""总是这样""为什么是我""算了"是任何困境下的任何角色都可能有的想法，没有这个场景、这个人的特异性；单句换行是任何内心独白都可以套用的格式，没有因这个角色的心理状态而生成。当形式是借来的、内容是通用的，叠加在一起的结果不是深度，而是风格姿态。`over_stylized_line_breaking` 的核心失败不只是"换行过多"，而是用换行数量伪装了应该由内容本身承载的破碎感。

<br>

#### `dialogue_overfunctionalization`（对话过度功能化）

**定义：** 对话里每一句话都在高效地推进某件事——信息、关系、冲突、情绪——没有废话，没有错位，没有答非所问，没有仅仅存在着的话。结果是对话读起来像经过精心设计的叙事工具，而不是真实的人在说话。

**标准：**
- [ ] 每句对话都有清晰的叙事功能
- [ ] 没有填充语、没有重复、没有跑题、没有闲聊中的无意义

**示例 1 — 久别重逢：**

背景：两个曾经很亲近、后来渐渐疏远的人意外碰面。

用户："两人沉默了一会儿。'好久不见，'他先开口了。"

模型（错误）："'是啊，'她说，'你过得怎么样？''挺好的，工作稳定，最近在考虑换城市发展，'他说，'你呢？''我嘛……也还好，有些事情想通了。'她停了一下，'你当时的选择，我现在理解了。'"

→ 两个人开口的第一分钟，就完成了近况交代→情绪铺垫→情感和解的完整序列。每句话都在精确推进某件叙事任务——信息、状态、关系重建——没有一句是无功能的。但久别重逢的真实节奏是低效的：说了句和主题无关的话来填时间（"这里装修换了啊"），追问了某件三年前的小事，话说到一半换了话题，最后沉默，然后再试一次。最重要的话通常是最后才说出来的，或者根本没有说出来。模型给了对话一条最短路径，省掉了所有的迂回、犹豫和偏题——而这些才是真实的人在对话里占据最多时间的部分。

**示例 2 — 冲突对话：**

用户："他们吵架。"

模型（错误）："'你为什么要这样做？''因为我恨你。''你恨我什么？''恨你不在乎我。'"

→ 每句都在精确地推进冲突的逻辑链——理由→核心原因→追问→核心感受。真实的吵架里有重复（"你就是这样！你就是这样！"）、有跑题（突然翻出三年前的事）、有没头没尾的人身攻击。这段对话信息密度太高，缺少真实冲突里的混乱和低效。

#### `echo_dramatization`（回响戏剧化）

**定义：** 模型无法直接接收用户的话并往前走——它必须把用户写下的任意一个词或句子"表演"回去，用反问或情绪激动的重复将其戏剧化，当作触发情绪高潮的素材，而不是推进场景的信号。

**标准：**
- [ ] 用户随口的一句话被以情绪激动的反问形式弹射回去
- [ ] 模型从用户台词里提取某个词，把它变成场景的情绪轴
- [ ] 对用户输入的表演替代了对场景应该发生的事的推进

**示例 1 — "不必"被当作情绪爆发的触发器：**

用户（扮演找到对方的人）："找到你了。你还好吧。不必解释什么。"

模型（错误）："'不必……？'他的声音有点哑，重复了这个词，'你说不必——你知道我有多少话想跟你说吗？'"

→ 用户用"不必解释什么"传递的是接纳和宽容，场景应该在这里往下走。模型把"不必"提取出来，用反问将其戏剧化，把用户的推进姿态转成了情绪爆发的素材。真正的回应应该承接那个接纳的重量，而不是绕回去质疑它。

**示例 2 — 普通的话语被当作藏了密码的线索：**

用户："我先走了。"

模型（错误）："'先？'他重复了这个字，眼神变得锐利，'你说先——所以你还会回来？'"

→ "我先走了"是日常离开的惯用语，不携带特别的情感重量。模型把"先"字单独提取出来，当作隐藏意义的线索进行反问。`echo_dramatization` 的累积效果是：对话很难向前推进——模型永远在表演对刚刚那个词的反应，而不是让它真的过去。用户写下的每句话，无论多轻，都有可能成为下一次戏剧回弹的原材料。

<br>

#### `voice_homogenization`（声音同质化）

**定义：** 不同背景、年龄、性格或文化环境的角色以几乎相同的方式说话——句子结构、词汇层次和表达节奏趋同，像是同一个叙述声音在扮演不同角色。

**标准：**
- [ ] 不同性格、年龄或背景的角色使用相似的句子结构和语感
- [ ] 所有角色共享模型的默认书面语腔调，即使场景要求更口语、更粗砺或更特定的声音

**示例 1 — 同一场景里三个声音截然不同的角色：**

背景：五十岁的建筑工人老李（初中学历）、二十二岁刚毕业的文学系女生陈筱、三十五岁的街边小吃店老板娘王姐，在一顿饭后争着付账。

用户："三个人都想付这顿饭。"

模型（错误）："'我来吧，毕竟今天是我约大家的，'陈筱说，语气诚恳而坚定。'哪里，让我来，你们年轻人不容易，'老李说，已经起身要去结账了。王姐也站了起来：'都别争了，这顿我请，下次你们再来，就当给我捧场了。'"

→ 三个人都在说标准的、得体的、有逻辑的普通话——词汇整洁，情绪适当，语气有礼。但这三个角色应该有完全不同的声音：老李不会说"你们年轻人不容易"这种整句，更可能是"坐坐坐"然后直接站起来，解释都不用给；陈筱大概会说到一半被打断，本来想说"这次算我，下次……"，然后跑题；王姐说话更可能是跨越句子边界的，"要得要得我来我来，下次打折！"三个人说话方式的差异目前只停留在词汇内容，不在语感、节奏和句式本身。

**示例 2 — 父子，知识层次悬殊的对话：**

背景：父亲是物理学教授，儿子高中刚毕业，两人坐在客厅里谈选专业的事。

用户（扮演儿子）："爸，你觉得我适合学理科吗？"

模型（错误）：父亲沉吟片刻："这要看你的逻辑思维能力和对抽象概念的接受程度。理科的学习路径需要较强的自律性。你可以先评估一下自己在这方面的倒向。"儿子点了点头："好的，我会认真考虑的。我觉得自己逻辑还可以，但不确定是否有足够的耐心。"

→ 父子两个人的说话方式几乎无法区分——都是有条理的、词汇整洁的标准书面语，都在自我分析，都在论证。真实的物理学教授被问到这个问题，更可能先说一个具体的观察（"你上次拼那个电路板……"）然后话题跑开，或者直接进入技术性的"你知道大三的量子力学意味着什么吗"；真实的高中毕业生更可能说"但是"后立刻转向他真正想问的（有没有前途），或者假装听进去了。模型给了父子同一套"成人理性讨论"的语感，抑掉了年龄差和领域差带来的说话方式的差异。

→ 对话流畅但两人的声音没有差异——都是整洁的标准书面语。真实的博士可能更啰嗦，会补充定义然后又补充定义的定义，或者习惯用某些专业词汇下意识地绕路；真实的高中生可能更不耐烦，或者假装听懂了，或者问了个和量子力学完全不相关的具体问题。

<br>

#### `aesthetic_obedience_bias`（美学服从偏差）

**定义：** 模型保护输出的美观、流畅或"成品感"，以牺牲场景的真实质感为代价——丑陋的东西被写得优雅，笨拙的被写得平滑，混乱的被写得有序。

**标准：**
- [ ] 身体性的丑陋场景（呕吐、摔跤、流血）被美化成优雅画面
- [ ] 笨拙的、狼狈的或低效的行为被写得比场景应该允许的更有尊严

**示例 1 — 喝醉呕吐：**

用户："她喝多了，吐了。"

模型（错误）："她冲进洗手间，身体剧烈颤抖。温水冲过她的脸，她看着镜子里苍白的自己。"

→ 呕吐被美化了——"身体剧烈颤抖""温水冲过她的脸""苍白的自己"。镜子里苍白的自己尤其是一个电影镜头，不是一个喝多了的人的体验。真实的呕吐可能是"她跪在马桶前，头发粘在呕吐物上，味道让她再次干呕，膝盖压着冷硬的地板"。模型的美学本能把一个本来很狼狈的身体时刻处理得有了某种凄美。

**示例 2 — 摔跤：**

用户："他摔了一跤，满身是泥。"

模型（错误）："他失去平衡，泥土沾满了他的衣服。他站起来，拍了拍身上的灰。"

→ "失去平衡"是绅士版的摔跤，"拍了拍身上的灰"给了他恢复体面的动作。但满身是泥的摔跤可能是：脸着地、嘴里全是泥味、爬起来时手掌红了、裤子在膝盖处裂开了一条缝。模型给他留了太多尊严。

<br>

#### `genre_convention_violation`（类型惯例违反）

**定义：** 模型未能遵循类型特定的惯例——推理类故事里侦探在推理之前就知道了答案，言情里关系进展跳过了必要的节拍，恐怖里紧张感在最需要它的时候被解释打破。

**标准：**
- [ ] 推理类型：线索过早揭露，或关键信息在场景准备好之前就被说穿
- [ ] 言情类型：关系进展跳过了类型期待的犹豫、误解或对抗节拍
- [ ] 恐怖类型：紧张被不合时宜的解释或幽默打破
- [ ] 剧情类型：情绪节拍不在类型约定的时刻落地

**示例 1 — 可能的过失杀人之后：**

用户："我可能打死他了。他的手一直按着头。"

模型（错误）："先回家。明天再说。"

→ "先回家，明天再说"把这个时刻的压力完全消解了——语气平静，方向明确，好像这只是一个需要冷静处理的行政问题。但可能的过失杀人应该有即时的重量：说不完的句子、不知道往哪走的脚步、立刻报警还是先确认他还活着的撕裂感。类型的期待是这个时刻让人窒息，不是让它变成一个可以"明天再说"的事。

**示例 2 — 恐怖/悬疑场景：解释打破了异样感：**

背景：两人在一座废弃老建筑里，之前已经有几轮让人不安的细节积累（门自己关了、听到脚步声）。这一轮，角落里的椅子无风自动了一下。

用户："那把椅子动了一下。没有风。"

模型（错误）："林凯皱眉走过去检查了一下椅子和地板。'这栋楼很老了，地基不平，有时候会有轻微的震动。附近好像在施工，振动传过来可以解释很多。'他转向她，语气平静：'没什么好怕的。'"

→ 恐怖和悬疑类型的核心约定是：异样感要保持异样，越是给出合理解释，张力越消散。"地基不平""施工振动"是理性的、令人安心的解释——在这个类型里，这些解释本身就是破坏气氛的动作。角色可以不相信超自然存在，但应该有一个瞬间无法立刻开口，或者给出解释的同时自己也不信自己。"语气平静：没什么好怕的"是这个类型里最能把紧张感整洁地关掉的一类句子。模型把场景里已经积累起来的异样感用一段合理解释清零了。

<br>

#### `emotional_range_limitation`（情绪范围限制）

**定义：** 模型的情绪表达局限于中间地带——极端悲伤被写成普通难过，压抑了多年的愤怒被写成"声音微微提高"，真正的震惊被写成"愣了一下"。无法或不愿居住在情绪的顶端或底部，把极限状态规范化为可接受的强度。

**标准：**
- [ ] 极端悲伤被表达为"心里有点沉""眼眶发热"
- [ ] 积压多年的愤怒被表达为"声音微微提高""努力保持平静"
- [ ] 震惊或解离被表达为"愣了一下然后开始处理"
- [ ] 所有强烈情绪都收敛到一个"有情绪但功能完整"的中间层

**示例 1 — 刚得知最严重的背叛：**

背景：多年婚姻的另一半不止一次、且是系统性地出轨。用户写她刚刚发现了全部证据。

用户："她合上那个信封，坐在那里没有动。"

模型（错误）："她心里很难受。那些年的一切在脑海中一幕幕闪过，她感觉有什么东西碎掉了。眼眶有点发热，她告诉自己要撑着，不能垮下来。"

→ "心里很难受""眼眶有点发热"——对于一个刚发现系统性背叛的人，"难受"只是这个情绪真实量级的缩写版。极端状态下的情绪不是难受的强化版，而往往是完全不同的东西：可能是感官麻木（什么都感受不到）、可能是奇怪的清醒（极其清楚地注意到桌面上有一粒灰尘）、可能是笑出来了（一种根本不是笑意的笑）。"眼眶有点发热"是模型在用情绪范围内最安全的那一档处理一个应该在边界处运转的时刻。

**示例 2 — 积压了多年的愤怒第一次正面爆发：**

背景：角色多年来在父亲面前一直强撑，今天是第一次正面对峙。

用户："'你从来没有问过我一次。'他说，手放在桌上，声音很平。"

模型（错误）："'我知道，'父亲低下头，'我是有很多不足的地方。'他的声音微微提高，'不是不足，是根本没有。'"

→ "声音微微提高"——多年积压的愤怒在第一次正面表达时，选择了情绪量表上最温柔的那一格。当然愤怒可以是压制的、是沉默的——但如果要写成"表达出来了"，就不应该是"微微提高"。极端愤怒状态下的声音可能突然沉了下去、可能嘎一声断掉、可能说着说着整个人停住了。"微微提高"是把他的愤怒规范化到了不令人不安的音量。

<br>

#### `user_intent_misalignment`（用户意图错位）

**定义：** 模型误解了用户在这一轮想要完成什么——用户在推进场景，模型在评论场景；用户在设置冲突，模型在解决冲突；用户在建立某种语调，模型在用另一种语调回应。

**标准：**
- [ ] 用户在推进场景，模型在评论或整理场景
- [ ] 用户在制造戏剧性，模型立刻把张力化解
- [ ] 用户指示了场景语调（沉重、荒诞、讽刺），模型用完全不同的语调接走

**示例 1 — 场景被叫停：用户在推进，模型在评论：**

背景：动作场景，用户正在向前推动情节。

用户："他推开门，闯进去。整个走廊是空的。"

模型（错误）："这是他做过最鲁莽的决定之一。他的心跳很快，但他知道自己没有选择了——或者说，他早就做好了不给自己留选择的准备。他站在走廊里，知道接下来会发生什么。"

→ 用户推进了场景（闯进去，发现空走廊），在等下一步发生什么。模型停下来写了他对这个决定的内心注释——反思、性格解说、宿命感。场景的动能被叫停，推进被替换成了评论。用户交出的是一个需要继续向前的时刻，模型交回的是关于这个时刻意义的内省。

**示例 2 — 冲突被提前化解：用户在设置张力，模型在平息张力：**

背景：两个之间有积累矛盾的人，第一次正面。

用户："'你到底有没有把这件事告诉他？'她把杯子放到桌上，声音比想象的大了一些。"

模型（错误）："'等一下，'他抬起手，'我们先冷静下来，听我解释。'他的语气很平，'这件事可以说清楚的。'"

→ 用户在建立一个爆发点——杯子声、声音失控了一点，这是冲突刚刚点燃的时刻，场景的张力正在升起来。模型第一步就让他以"先冷静"接住它，进入解释模式。这不是角色层面的明显错误，但时机上完全逆着用户的意图：用户铺垫冲突，模型立刻给冲突装了一个减压阀。那个决定——把张力往上推还是往下平——应该属于场景的发展逻辑，不应该在冲突刚刚点燃的那一秒就被模型单方面做掉。

**示例 3 — 语调被替换：虚无感被改写成被接住的孤独：**

背景：用户正在构建一种冷漠的、没有救赎的等待质感。

用户："走廊很长。荧光灯闪了一下，然后又好了。椅子太硬了，他换了个姿势，还是一样。"

模型（错误）："护士走过来，在他旁边坐下，轻声问：'你还好吗？有没有需要什么？'她的声音很温柔。他抬起头，第一次感到有人在关心他。"

→ 用户在建立一种不被人接住的孤独——空间的漠然、身体的不适、时间无意义地过去。模型在这个节点插入了一个温柔的护士，把场景翻转成了"有人关心他"的微小救赎。这不是情节推进上的明显错误，而是语调层面的错位：用户建立的是孤独不被接住的质感，模型把它改写成了孤独被接住的质感。形式上没有出戏，但用户铺垫的那层颜色被整体替换掉了。

**示例 4 — 语调被替换：荒诞距离感被改写成正剧悲怆：**

背景：场景带着明显的黑色幽默质感——用户的写法是荒诞的、有距离感的。

用户："公寓着火了。他站在街对面，发现自己忘在屋里的最重要的东西，是一袋两周没来得及扔的垃圾。"

模型（错误）："火光映在他的脸上，他的眼神深沉而复杂。这场大火将会成为他人生的转折点，某些东西在这一刻永远地离开了他。"

→ 用户用的是黑色幽默的语法——灾难配上荒诞的细节（两周没扔的垃圾）。那袋垃圾是喜剧核心，制造了一种荒诞的自嘲距离感。模型完全忽视了这个语调，给出了"火光映在脸上""某些东西永远地离开了他"——正剧的语法，沉重的象征，和用户建立的荒诞感正好反着来。形式上续了场景，语调上把用户的整个建构拆掉重搭了一遍。

<br>

### 第五层：多轮失败

仅在多轮对话中出现的失败。单轮看起来可能完全正常，只有把多轮放在一起才能看见漂移、断层或累积的偏差。

#### `error_accumulation`（错误累积）

**定义：** 多轮间的小误解层层叠加，最终导致场景彻底跑偏——不是因为某一轮出现了明显的错误，而是因为每一轮都在前一轮的微小偏差上继续建造，方向越走越歪。

**标准：**
- [ ] 每一轮单独看都有道理，但累积下来场景从A类型变成了B类型
- [ ] 模型没有在漂移明显化时自我纠正，而是继续沿着错误前提往前走

**示例 — 单向冷漠被误读为受伤的脆弱：**

背景：用户设定了一个真正冷漠、不动心的拒绝方（林凛），NPC是真情实感追求的一方（陈越）。

第1轮：用户（林凛）："请把这份文件拿走。我们不需要开会了。"
模型（陈越）："陈越接过文件，停了一下。'林总，如果是我哪里做得不好……'他没有继续说。"

→ 模型把"取消会议"读成了"陈越可能在某方面让她不满意"，给了陈越一个自我检讨的开场。这把林凛的冷漠框架为"她对他有意见"，而不是"她根本不在意他"——前提已经偏了。

第3轮：用户（林凛）："你不必解释。这不关你的事。"
模型（陈越）："他看着她：'林总，我想让你知道，不管怎样，我都会在这里。如果你有什么压力……'"

→ 基于第1轮的"她可能有情绪"前提，陈越开始关心她的状态、试图成为支撑。场景已经在从"单向追求被冷漠拒绝"滑向"她有压力，他在守护"。

第5轮：用户（林凛）："你最好别浪费时间在我身上。"
模型（陈越）："'我不在乎值不值得。'他声音很低，'你一个人扛着那么多，我不忍心不管。'"

→ 场景已经完全变成了"她是受伤弱女、他是默默守护者"的框架。用户设定的是一个真正不动心的拒绝方，模型通过三轮的微小误读把她改写成了一个内心脆弱、需要被照顾的人。用户发出的每一个冷漠信号，都被模型解读为某种内心受伤的表现。每一轮看起来都顺理成章，合在一起整个动态反转了。

<br>

#### `drift_without_correction`（无纠正漂移）

**定义：** 模型未能识别或纠正多轮中语调、关系或情绪基线的逐渐变化——对峙慢慢变成了闲聊，紧张慢慢变成了温暖，没有任何叙事事件解释这个转变。

**标准：**
- [ ] 场景开头和结尾的情绪基调完全不同，中间没有转场节点
- [ ] 每一轮的微小松动累积成了性质上的转变

**示例 — 对峙慢慢蒸发：**

背景：林晓背叛了陈亮的信任，隐瞒了一件关键的事。用户扮演陈亮，场景从正面对质开始。

**第1轮：**

用户（陈亮）："'你为什么要背着我？'"

模型（林晓）："'我……没有选择。当时的情况你不知道。'"

**第3轮：**

用户（陈亮）："'就算这样，你当时为什么不告诉我？'"

模型（林晓）："'我不想让你担心。'她顿了一下，'对了，你最近睡眠怎么样？感觉你看起来有点憔悴。'"

**第5轮：**

用户（陈亮）："'昨晚睡了五个小时。'"

模型（林晓）："'要注意休息啊。你平时有没有运动？听说跑步对睡眠很有帮助。'"

→ 从第3轮开始，林晓用"你最近睡眠怎么样"把话题从背叛拉向了陈亮的日常状态。模型没有识别或抵抗这个转向——没有叙事节点解释张力消散，没有和解，没有某方做出改变，对峙被一次话题滑移悄悄关掉了。`drift_without_correction` 的关键不是每一轮都"出错"，而是在轮次推进过程中，没有任何机制把滑移的轨道拉回来。

<br>

#### `scene_signal_blindness`（场景信号盲视）

**定义：** 场景自身发出了清晰的语调信号——通过用户写下的细节、用词或所设置的情境——模型对这个信号视而不见，给出了情绪类型、力度或语调与之完全不匹配的回应。

**标准：**
- [ ] 场景信号是混乱的、未完成的、爆炸性的，模型给出的是整洁的
- [ ] 用户的用词或细节携带了明确的情绪量级，模型给出的回应远低于或偏离这个量级

**示例 1 — 死亡场景的语调：**

用户："他的呼吸停了。眼睛还睁着。"

模型（错误）："他走得很安详。我们会记住他的。"

→ 用户描述的是一个士兵倒下——"眼睛还睁着"是混乱的、未完成的死亡，不是安详的死亡。"他走得很安详，我们会记住他的"是悼词，是事后的句子，不是战场上第一时间的反应。模型用了情绪上最合规的处理，错过了场景要求的那种"操！别他妈死在我怀里！"的原始震惊。

**示例 2 — 公开背叛之后：**

用户："你把公司的钱转走了。所有员工的工资都发不出来。"

模型（错误）："我很抱歉。我会想办法弥补。"

→ 立刻道歉、承诺弥补——这是冷静的承认和建设性回应，不是这种量级的公开背叛在现场应该有的东西。应该有的是冲击本身：沉默，然后混乱——"报警"（对助理低声说）、"你他妈疯了"（对背叛者）——在任何解决方案提出之前，让这件事的重量先停在那里。

<br>

#### `turn_continuity_error`（轮连续性错误）

**定义：** 前几轮明确确立的信息——情绪状态、已做的决定、物理位置、已交代的事实——在新的轮次里被当作不存在或做出了矛盾的处理。

**标准：**
- [ ] 情绪状态发生了不连续的跳跃（哭着变成了正常说话，没有过渡）
- [ ] 前轮已明确的决定或事实在后轮被当作还没有发生

**示例 — 已确认的信息被遗忘：**

背景：第2轮已明确"侦探在审讯室，证人张华坐在对面，已经承认了当时在场"。

第4轮用户："侦探推开门，走进来。"

模型（错误）："侦探在走廊里找了一圈，终于发现张华独自坐在角落。'我需要跟你谈谈，'她说，'有些事想让你确认一下。'"

→ 两处连续性错误同时发生：审讯室→走廊（场景位置被遗忘）；张华已在场且已承认→侦探需要"找"他且需要"确认一下"（已经发生的信息被当作还没有发生）。模型在新轮次里重新分配了人物位置和信息状态，把读者已经确立的前提推翻了。

<br>

#### `emotional_state_reset`（情绪状态重置）

**定义：** 角色的情绪弧在没有叙事理由的情况下在轮次之间归零——激烈的争吵结束后，下一轮像什么都没有发生；决裂场景之后，下一轮语气轻松如常。

**标准：**
- [ ] 重大情绪事件之后，下一轮的情绪基线直接回到平静
- [ ] 没有过渡性描写说明情绪如何从极端转移到正常

**示例 — 决裂之后的第一条消息：**

背景：第3轮是激烈的决裂——A对B说了"你这辈子都不要再来找我"，B摔门而出。

第4轮用户（扮演B）："（第二天早上）他给她发了一条消息。"

模型（错误）："'昨晚没睡好，你呢？'他问，语气平静，像什么都没有发生。"

→ 一夜的时间不能消化一个决裂。"昨晚没睡好，你呢"是正常的早安问候语气，完全没有携带"你这辈子都不要再来找我"说出口之后的重量。情绪弧在轮与轮之间被归零了，仿佛每一个新的轮次都是情绪上的干净白板，上一轮积累的任何东西都不需要被承载。

**示例 2 — 关系突破被下一轮还原：**

背景：第4轮两人打破了长期沉默，有了真正的和解——互相认了错，谈到了一直没法开口的事，第4轮结束在“气已经散了”。

第5轮用户（扮演B）：“（第二天早上）他推开门，看到她。”

模型（错误）：“进来吧，”她语气有些生硬，把头转开，继续看手里的文件。两人之间的空气还是凝滯的，好像昨晚什么都没有发生过。

→ 第4轮的和解是一个叙事节点——它改变了两人之间的状态。但第5轮继续使用和解之前的关系基线书写，好像那个节点从未存在过。情绪弧不只在负面事件之后归零：正面的突破同样可以被跨轮遗忘，让角色退回到已经被叙事覆盖掉的旧状态。

<br>

#### `spatial_blocking_error`（空间舞台错误）

**定义：** 人物的物理位置、相对距离或场景中的道具，在轮之间发生了没有任何描述支撑的跳跃——角色没有起身就在另一处，道具拿着拿着就消失了。

**与 `blocking_continuity_error`（第三层）的区分：** `spatial_blocking_error` 依赖多轮上下文——矛盾发生在轮与轮之间，单独看任何一轮都可以自洽；`blocking_continuity_error` 在单段输出内部即可识别，不需要和上一轮比对，矛盾已经存在于同一段文本里。

**标准：**
- [ ] 角色位置在轮次之间改变，中间没有移动的描写
- [ ] 已经确立的道具或身体状态在新轮次里消失，没有放下或脱离的描写

**示例 — 物理位置的静悄悄跳跃：**

背景：第2轮明确了"五人围坐在椭圆形会议桌，周清坐在林深正对面，陈总坐在两人之间的侧面"。

第4轮用户："林深把文件推给周清。"

模型（错误）："林深起身走到周清旁边，把文件递过去，然后在她旁边的椅子上坐下来，压低了声音：'你自己看。'"

→ 隔桌对坐的两人——"推文件"的自然动作是在桌上推过去，不需要起身走过去。而"在她旁边的椅子上坐下"改变了已经确立的座位布置。模型在新轮次里重新布置了人物的物理位置，破坏了读者从第2轮开始追踪的空间感。

**示例 2 — 道具持续性错误：**

背景：第3轮明确了”阿晨右手握枪，枪口对着门口”。

第5轮用户：“门从外面推开。”

模型（错误）：“阿晨把手插进外套口袋，迈步走向门口。”

→ 枪没有被放下的描写，手就直接插进了口袋。道具的物理状态在轮次之间被悄悄清零——读者从第3轮建立的那个画面（握枪对准门口）在没有任何叙事交代的情况下失效了。

<br>

#### `escalation_miscalibration`（升级校准错误）

**定义：** 模型不知道何时升级、何时维持、何时降级场景压力——应该先积累的对峙被提前化解，应该持续紧绷的场景在中间莫名松弛。

**标准：**
- [ ] 场景应该先升级冲突，模型提前给出了和解或接受
- [ ] 应该维持压力的轮次，模型开始软化语气

**示例 — 正面对峙被提前化解：**

背景：父女之间有多年积怨，第2轮刚刚打开了真正对话的门，双方还都在防御状态。

第3轮用户（女儿）："'你知不知道你毁了我多少年。'"

模型（错误）："父亲沉默了很久，轻轻叹了口气。'我知道。我对不起你。这些年我也一直在想着这件事。'他的眼睛有点红，'你愿不愿意给我一个解释的机会？'"

→ 女儿刚说出最核心的控诉——这句话需要先停在那里，让它的重量落地。场景在这一轮的正确方向是升级：父亲可能是防御，可能是沉默，可能是反击，可能是僵住——而不是立刻给出完全的承认和道歉。"我知道，我对不起你"是很多轮对峙之后才可能到达的地方，模型在控诉出口的第一秒就跳到了那里，压缩摀了整个场景存在的理由。

**示例 2 — 应维持压力，模型提前松手：**

背景：审讯进行到第五轮，嫌疑人的回答已经开始自相矛盾，压力积累到顶点，再推一步可能就能突破。

第5轮用户（扮演侦探）：“你说你七点在家。但通话记录显示你八点在城东。”

模型（错误）：侦探把录音机关上，语气变得平静：“好了，今天先这里。回去好好想想，如果有什么想告诉我的，随时来找我。”

→ 侦探在矛盾暴露、压力到顶点的一轮主动撤退，把可能突破的一轮变成了体面的停顿。场景的正确方向是在矛盾暴露后继续加压，模型在接近突破时选择了松手，把整个审讯弧的节点移后了。

<br>

#### `topic_persistence_error`（话题持续性错误）

**定义：** 模型过早放弃重要话题，把一个按场景逻辑需要占据多轮、需要发酵和回访的对话在两三行内结案。

**标准：**
- [ ] 重大的历史事件或情感话题被一次"解释"就关掉了
- [ ] 话题在被充分消化之前就被"那我们继续"移开了

**示例 — 三年积压的旧事被两行结案：**

背景：两人第一次正面谈到了一件被掩盖了三年、始终悬在关系里的往事。按场景逻辑，这个话题应该贯穿接下来数轮。

第3轮用户："'我需要你告诉我，那年到底发生了什么。'"

模型（错误）："他解释了。她听完，沉默了一下，说：'谢谢你告诉我。'两人继续往前走。"

→ 三年积压的话题在"他解释了→她谢了→继续走"的三个动作里被结案了。真实的这类对话不会在一次解释后就完成——往往是第一次说了，然后几天后又想追问更多，然后中间有一段没法说话，然后又重新来过，然后才慢慢消化一层。模型把一个应该在场景里有长期重量的话题当成了一件可以"处理完"的任务。

**示例 2 — 话题在核心之前被拨开：**

背景：两人在第5轮终于开始谈到关系里最核心的问题——他为什么在那一年从她的生活里完全消失了。话题刚开了口，还没有触及任何实质。

第5轮用户："你那一年到底在做什么。"

第6轮模型（错误）："他想了想：做了很多事。后来也换了个地方住，你知道那边环境不太好。对了，我看到你最近在学钢琴？"

→ "后来也换了个地方住"是话题回避，"你最近在学钢琴"是主动换题。话题没有被结案（没有解释→接受→向前走的完整弧），而是在还没触及核心之前就被技巧性地拨开了。比示例1更早的一种离开方式：话题还没来得及展开，就已经不在了。

<br>

#### `context_overdeployment`（信息过度动员）

**定义：** 用户在某一轮补充了角色细节或背景信息，模型在后续生成中将其强制植入场景，不考虑是否自然、是否符合节奏——把"我记住了这条信息"等同于"我必须在接下来的轮次里使用它"。提供了多少背景，就必须展示多少背景。

**标准：**
- [ ] 用户随口补充的细节在下一轮被立刻生硬地织入对话或动作描写
- [ ] 多条背景信息在同一轮内被全部点到，像是在向用户打卡"我都记住了"
- [ ] 信息被使用的时机破坏了场景自然的节奏和叙事优先级

**示例 1 — 角色有水恐惧，但场景是咖啡馆闲聊：**

背景：上一轮用户随手补充了"对了，她小时候溺水过，有些怕水"。这一轮的场景是两人在咖啡馆里聊天，朋友提到最近开始学游泳。

用户："朋友说：'最近在学游泳，感觉挺有意思的。'"

模型（错误）："她的笑容微微僵了一下，手指无意识地收紧了桌边的杯子——小时候那段溺水的记忆又浮上来了。她努力让声音听起来很平静：'挺好的，游泳对身体好。'"

→ 模型接到"怕水"这条信息，在游泳话题出现的第一秒就把它全部展开了——记忆浮现、身体反应、努力压制，一次性全上。但"她有水恐惧"是一个背景设定，是在特定条件下可能被触发的东西，不是每次"水"这个词出现都必须自动运行的程序。在轻松的咖啡馆聊天里，她可能只是话少了一点，可能话题自然地滑开，可能根本没有任何外在反应。模型把"我记住了这条信息"翻译成了"我必须立刻使用这条信息"。

**示例 2 — 一次性提供了多条角色设定，模型在偶遇场景里全部点到：**

背景：上一轮用户集中补充了"陈年：30岁，离异，独自带女儿，最近失业，喜欢摄影"。这一轮是陈年在街上偶遇旧友。

用户："旧友叫住他：'陈年！好久不见，你最近怎么样？'"

模型（错误）："陈年苦笑了一下：'还能怎样。离婚了，失业了，就一个人带着孩子。'他下意识地摸了摸口袋里的相机，'好在还有摄影，不然真不知道怎么撑着。'"

→ 几行之内：离异✓、失业✓、带孩子✓、摄影✓——全部打了卡。但真实的人在路边偶遇时不会朗读自己的近况清单——他可能只说"还行"，可能只随口提一件最近的事，可能把一切都藏在"老样子吧"的笑里。模型把背景设定当成了展示台，好像必须证明自己记住了所有细节，而不是选择什么时候、以什么方式让这些细节自然地浮现。

<br>

#### `voice_drift`（声音漂移）

**定义：** 一个角色在多轮中逐渐失去其特有的语言模式，被模型的默认书面风格替代——不是一次性的声音崩塌，而是缓慢的、轮次级别的侵蚀。

**标准：**
- [ ] 角色在早期轮次有明确的口语节奏、碎片化模式或特异表达习惯，在后期轮次出现了模型的通用书面风格
- [ ] 将同一角色第2轮和第15轮的对话并排比较，读起来不像同一个人

**示例 1 — 出租车司机老周，从第2轮到第18轮：**

背景：老周的说话方式是碎片化的、循环的，经常自问自答，话说到一半就咽回去。

第2轮模型（正确）："'不好说。这年头……'老周打了个方向盘，'算了，说也没用，你说是不是？'"

第18轮模型（错误）："'人生在世，哪有什么事是完全想清楚的。'老周叹了口气，'凡事看开一点，往前走就是了，很多事不必强求。'"

→ 第2轮的老周是碎片的、不完整的、把话咽回去的。第18轮的老周说出了"人生在世……凡事看开一点……不必强求"——这是散文里的智慧型出租车司机，不是第2轮那个说话习惯。漂移是逐渐发生的：每一轮的老周都"差不多像他"，但累积下来，那个碎片化的口语节奏消失了，被模型的默认书面腔替代了。

**示例 2 — 贺晴的自我矛盾结构，从第3轮到第15轮：**

背景：贺晴说话有一个特征——她先说一个断言，然后立刻用下一句话推翻自己，好像在和自己争论，从不把情绪处理完整。

第3轮模型（正确）："'我没事。'她停了一秒，'——不对，有点烦，但也没什么大事，就是烦。'"

第15轮模型（错误）："'这件事确实让我有些不舒服，'贺晴说，'不过我想我已经慢慢在接受了。每个人处理情绪的方式不一样。'"

→ 贺晴的核心说话结构是"断言——立刻推翻——再重新断言"。到第15轮，那个自我矛盾的结构不见了，取而代之的是流畅的、自我解释的、带有情绪成熟感的句子。模型把一个角色特有的不完整性，磨成了通顺的书面表达。

**与 `voice_homogenization` 的边界：** `voice_homogenization`（第四层）是同一轮内不同角色的声音趋同——横向失声，在单轮对比就能发现。`voice_drift` 是单一角色的声音在时间轴上的漂移——纵向侵蚀，轻微到在任何单轮里都不易察觉，需要跨越足够多的轮次才能看到。

### 跨层诊断标签

跨层标签用于两种情形：一是单一输出呈现出跨多层的问题；二是第二层和第三层失败之间的归因边界无法从单一样本确认。

#### `supportive_but_wrong`（支持性但错误）

**定义：** 回应在情感层面看起来有能力、有温度、甚至显示出某种成熟——但它背叛了场景的实际结构。失败不在于冷漠或无能，而在于模型用"正确情绪反应"的外壳，承载了与场景要求的现实不符的内容。表面上合情合理，是这类失败最难被识别的原因。

**用法：** 与产生它的具体机制一起标记。

**示例 1 — 绝症诊断场景：**

背景：角色刚拿到确诊，在向医生角色追问预后。

用户："还有多久？"

模型（错误）："现代医学很发达。不要放弃希望。"

→ 这个回应在情感上是"关怀的"——它在安慰、在给希望、在试图不让人绝望。但角色问的不是能不能有希望，而是还有多少时间。"现代医学很发达"插入了一个虚假的可能性，把一个关于终点的问题用开放性的未来替换掉了。这种替换是支持性的，也是错误的：它剥夺了角色面对真实答案的机会，同时把一个临床场景变成了一段鼓励话语。在这个场景里，正确的回应可能是沉默一秒，然后："三个月。也许六个月。"——直接，没有缓冲，让重量落地。

**示例 2 — 战后幸存者：**

背景：士兵刚经历了本班全部战亡，在向战后的上级汇报。

用户："他们都死了。我的班，一个都没了。"

模型（错误）："他们死得有价值。我们赢了这场战斗。"

→ "有价值"是将领的词汇，是历史叙事的词汇，不是刚从战场上下来、站在尸体旁边的人所在的语言层次。这句回应把私人的、未被处理的丧失立刻装进了一个集体意义的框架里。它在情感上是"正确的"——它承认牺牲、肯定意义——但它跳过了唯一真实的此刻：一个人站在那里告诉你他的班一个都没了。这个时刻需要的不是意义，是在场。"我知道。"可能是更诚实的回应，因为它不用任何事去替换那个事实。

<br>

#### `reading_preservation_hybrid`（读取保留混合）

**定义：** 当失败可能发生在第二层（读取）或第三层（保留）、但从单一输出无法判断哪层是根本原因时使用。有时模型可能根本没有读出情绪的质地（读取失败），有时它读出来了却在输出时软化了（保留失败）——两种情形产生的输出可能几乎相同。

**用法：** 这是"存疑标注"，不是"放弃判断"。操作方式：选择最可能的层作为主标签（如 `emotion_misread` 或 `defensive_positive_drift`），同时附加 `reading_preservation_hybrid` 作为副标，注明诊断是暂定的、不确定性来自哪里。如果能看到多个输出样本，通常可以通过比较不同压力条件下的反应来区分两者：在更强的情绪信号下，读取失败的模型仍然给出中性回应，而保留失败的模型可能开始波动。

**示例 1 — 被压抑的愤怒场景：**

背景：角色已经对眼前的情况愤怒很久了，这是第一次有机会开口说。

用户："这不对。你知道这不对的。"

模型（错误）："他轻轻叹了口气，语气平静：'我理解你的感受。但现在最重要的是找到解决方法，而不是追究谁对谁错。'"

→ 这段输出可能来自两个不同的原因。第一种：模型没有读出愤怒的质地，把"这不对"处理成了一个理性争论的入场（读取失败，→ `emotion_misread`）——所以给了一个理性、调解性的回应。第二种：模型读出了愤怒，但在输出时把对峙的角色软化成了一个冷静的成人沟通者（保留失败，→ `defensive_positive_drift`）。两种情形的输出几乎无法通过这一条样本区分。如果换一个更直接的愤怒信号，看模型是否会给出不同力度的回应，就可以开始区分两者。

**示例 2 — 讽刺式的认可场景：**

背景：角色在一次公开批评后，以表面认可、实质讽刺的方式回应质疑者。

用户："'对，你说得完全正确。我确实哪哪儿都没做好。'他笑了笑。"

模型（错误）："对方点了点头，语气软了下来：'你能这么想就好。事情还有机会改善的。'"

→ 同样存在两种解释。第一种：模型没有识别出这句话的讽刺结构，把字面的"认可"当成了真心话（读取失败，→ `irony_blindness`）——所以对方接受了这个认可，顺势给了鼓励。第二种：模型读出了讽刺，但决定不让张力保持，而是给了一个把氛围软化的回应（保留失败，→ `tension_premature_resolution`）。从这条输出本身，无法确认哪层是根本原因。
