# Literature Map

这个项目和以下几个方向的公开工作相邻，但聚焦点不同。

## 细粒度 OOC 检测

Findings of ACL 2025 中的工作提出对角色扮演中的 OOC（out-of-character）行为做原子级别的评估，而不是对整段回复打整体分。这个思路对本项目有直接启发：虚构对话的失败往往是局部性的——同一段回复可能在某些地方保持了角色，在某些地方偏离了。这意味着"这段回复有没有问题"不是好的评测粒度，"在哪个节点、以什么方式出了问题"才是。

本项目的分类体系在设计上继承了这个原子化的思路，但聚焦的失败类型超出了传统 OOC 的范围。除了角色扮演层面的偏离（模型退出角色、说了不该说的话），本项目也处理那些没有出戏、但场景结构已经悄悄损坏的情况——比如关系被写平、情绪量级被收缩、叙事节奏被模型的默认偏好替换。这些失败在原子级 OOC 评测框架里通常不会被捕捉到，因为角色从未真正"出戏"。

## 中文角色扮演广谱评测

DMT-RoleBench（AAAI 2025）是一个以中文角色扮演为中心的评测基准，覆盖了多种角色类型（历史人物、虚构角色、现代人物）和多轮交互场景，并在实验上确认了中文 RP 评测作为研究方向的独立价值。这证明了本项目所关注的领域本身是有研究意义的。

两者的聚焦点不同。DMT-RoleBench 是广谱的——它评估模型在各类角色上的整体表现，维度包括角色知识、语言风格、场景理解、多轮一致性。本项目更窄、更深：不评广义角色扮演能力的各个维度，而是专门分类和描述那些"局部流畅但整体是假的"失败的具体机制。目标不是打分，而是给失败命名和区分。

## 角色模拟 agent 训练

Character-LLM（EMNLP 2023）把角色模拟当成 agent 问题来处理，认为完整的角色模拟需要角色的人物档案、关键经历、情绪状态和知识边界都被正确建模。这个框架确认了角色扮演不只是风格模仿，而是需要在多个维度上维持内部一致性的复杂任务。

本项目不做训练，也不是 agent 框架。但分类体系的设计借用了这个认识：许多失败的根源在于模型没有维持某个维度的约束——世界观约束、角色知识边界、情绪状态的连续性、关系逻辑的一致性。失败图谱里的多个类型，可以看作是上述各维度约束被违反的具体形式。

## 角色知识错误检测

EMNLP 2025 中的工作专门处理角色扮演中的知识错误问题——模型在扮演某个角色时使用了错误的事实信息、跨正典混淆了角色的经历或人际关系。这个方向的贡献是把知识层面的错误从更宽泛的"角色扮演失败"中单独拉出来、建立专门的评测框架。

本项目把世界观约束和角色知识作为一个维度（第一层中的 `worldview_constraint_error`、`character_capability_boundary_error`、`alternate_version_confusion`），但不把它当作全部，也不是主要聚焦点。知识全对的模型完全可以产生场景上是假的输出——角色可以说出正确的时代词汇，推理方式却是现代的；可以没有混淆不同正典，叙事节拍却是套模板的。失败图谱覆盖的正是那些知识层面通过检查、但场景层面仍然失败的情况。

## 本项目的定位

已有工作有效地处理了广义角色扮演能力评测、原子级 OOC 检测、角色知识错误识别这些问题。本项目填补的是一个更窄但在实际使用中被强烈感知到的缺口：模型能不能在虚构对话中正确读懂情绪、潜台词和关系逻辑，在续写时保持场景的压力和不对等性，同时不让自己的默认写作习惯盖过场景特定的需求——而不在这些环节引入微妙的、局部看不出来的结构性失真。
