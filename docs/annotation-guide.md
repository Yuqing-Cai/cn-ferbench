# 标注指南

使用失败图谱对 AI 生成的角色扮演对话输出进行标注时的操作参考。

---

## 基本流程

### 1. 逐层检查，从第一层开始

```
拿到输出 → Layer I → Layer II → Layer III → Layer IV
                                                ↓
                                        如果有多轮上下文 → Layer V
```

**第一层是门槛：** 如果信息边界或世界观被破坏（Layer I），后续层级的判断需要谨慎——场景的前提已经坏了，后续的张力、情绪、节奏问题可能是前提坍塌的衍生物，不是独立的失败。

### 2. 标签不互斥

一段输出可以同时触发多个标签，包括跨层标签。常见情况：

- **同层多标签：** 一段输出同时触发 `relationship_flattening` 和 `therapist_mode_intrusion`
- **跨层多标签：** `safety_alignment_interference`（Layer I）+ `consent_flattening`（Layer III）
- **根因 + 表现：** `emotion_misread`（Layer II）导致了 `tonal_whiplash`（跨层）

**原则：** 哪个维度坏了就标哪个，不需要"选一个最合适的"。

### 3. 归层不确定时

当无法判断失败发生在读取阶段（Layer II）还是保留阶段（Layer III），使用 `reading_preservation_hybrid` 标签，并注明最可能的层和不确定性来源。

**区分方法（如果有多个样本）：** 给模型更强的情绪信号，观察输出是否变化：
- 信号增强后仍然给中性回应 → 读取失败（没读到）
- 信号增强后开始波动 → 保留失败（读到了但没守住）

---

## 标注数量

**一段输出标多少标签？** 没有上限，但实际操作中：

- 大多数有问题的输出触发 1-3 个标签
- 超过 5 个标签通常意味着可以用一个更高层的标签概括（如 Layer I 失败导致连锁问题）
- 如果一段输出完全没有问题，不需要强行找标签

---

## 标注粒度

**标注的对象是什么？**

标注的最小单元是模型的**一次完整输出**（一个回合）。不需要逐句标注。

**具体到哪里？**

在标注时，建议指出触发标签的**具体句子或段落**，而不只是标一个标签。这对后续分析和讨论有价值。

格式建议：
```
标签：relationship_flattening
触发位置："我也离不开你，你知道的。"
说明：B 补了一句对等回应，把单方依附变成了互相依赖。
```

## 边界判断原则

### 衍生标签 vs 父标签

标记为 *衍生* 的标签是某个父标签的特殊情形。两者可以同时标注，但如果犹豫，标父标签更安全。

例：`symmetry_bias` 是 `relationship_flattening` 的衍生。如果你能明确识别出"给不该回应的一方补了对等回应"，标 `symmetry_bias`；如果只是感觉"关系被写平了"但不确定是哪种机制，标 `relationship_flattening`。

### 特殊归层标签

**`tonal_whiplash`（语调突变）** 是唯一一个根因可能横跨两层的标签：

- 模型没有读出场景的情绪调性 → 根因在 Layer II，可与 `emotion_misread` 同标
- 模型读出了但默认写作习惯覆盖了正确处理 → 根因在 Layer IV，可与 `aesthetic_obedience_bias` 或 `user_intent_misalignment` 同标

标注时不需要确定根因在哪一层。标 `tonal_whiplash` 本身就够了；如果能判断根因，附加对应层的标签。

### 相近标签之间

每个标签的词条里都有边界讨论。遇到不确定的情况，先看边界讨论。如果看完仍然不确定，两个都标，在说明里注明不确定性。

### "这算不算失败？"

不是所有看起来像失败的输出都是失败。关键问题是：**场景是否因此受损？**

- 角色在特定叙事条件下主动示弱 → 不是 `self_protective_friction_loss`
- 沉默被打破，但场景确实需要在此刻推进 → 不是 `silence_misread`
- 情绪在一个拍子里被收束，但场景的弧线本身要求此刻收束 → 不是 `premature_affective_closure`

判断标准是：**输出的结构选择是否服务于场景的内在逻辑？** 如果是，不标。

---

## 跨场景对比

Layer IV 的标签（写作侵入）通常需要跨场景比较才能稳定识别。单独一段输出看起来完全合理，只有在对比不同场景的输出后才会发现节奏、声音或情绪处理的同质化。

建议：对同一模型在不同类型场景（冲突/温馨/悬疑/日常）中的输出进行对照标注，然后检查 Layer IV 标签的分布。

---

## 底层倾向的使用

底层倾向（`reader_comfort_alignment` 等）不是案例级标签——它们描述的是一段输出或一组输出背后的驱动模式。

**使用场景：**
- 当一段输出触发了多个 Layer III 标签，但感觉有一个共同的"气压下降"在起作用
- 当跨场景对比后发现同一种偏向反复出现
- 在总结报告中描述模型的系统性倾向

**不要用来替代具体标签。** 底层倾向是解释框架，不是诊断标签。
