# 项目联动方案：用失败图谱改进命运生成器

## 现状分析

### cn-failure-atlas
- **是什么：** 中文虚构对话失败模式分类体系
- **优势：** 5 层失败类型，覆盖前置条件、意义读取、场景保留、写作侵入、多轮失败
- **新增：** AI Companion 特定失败类型（治疗性语言、情绪劳动失衡、准社会关系强化等）
- **使用场景：** 质量评估、失败诊断、模型训练数据清洗

### oc-fate-generator
- **是什么：** 18 轴交互式 OC 设定与命运生成器
- **优势：** 多轴组合、内部逻辑完整、在线 Demo 可用
- **生成模式：** 开场模式（角色 + 开场场景）/ 时间线模式（完整命运骨架）
- **使用场景：** 角色设定、创作灵感、AI 陪伴产品角色库

---

## 核心洞见

**这两个项目是天然互补的：**
- **失败图谱 = 质量检查器**（识别什么是不好的）
- **命运生成器 = 内容生产者**（生成什么是好的）

**联动价值：**
1. 用失败图谱审计生成器的输出，确保不踩雷
2. 在生成器里内置"反失败模式"，主动避免常见问题
3. 把失败图谱变成生成器的"质量开关"——用户可以选"严格模式"（避免所有失败）或"创作模式"（允许某些失败以换取戏剧性）

---

## 具体改进方案

### Phase 1: 质量审计功能（1-2 小时）

**功能：** 在生成器里加一个"审计"按钮，用失败图谱检查生成的内容

**实现方式：**
1. 在 `app.js` 里加一个 `auditContent(content)` 函数
2. 内置失败图谱的检查规则（简化版，覆盖最常见的 5-10 种失败）
3. 生成后自动运行审计，显示"通过/警告/失败"

**示例检查规则：**
```javascript
// 伪代码示例
function auditContent(content) {
  const failures = [];
  
  // 检查治疗性语言侵入
  if (content.includes("和解") || content.includes("接纳") || content.includes("边界")) {
    failures.push({
      type: "therapeutic_language_intrusion",
      message: "检测到现代治疗性语言，可能不适合古代/奇幻背景"
    });
  }
  
  // 检查情绪劳动失衡
  if (content.includes("永远") && content.includes("等你")) {
    failures.push({
      type: "emotional_labor_imbalance",
      message: "AI 角色被设定为永远稳定，可能失去真实感"
    });
  }
  
  // 检查亲密度升级偏向
  if (content.includes("哪里都好") && context === "first_meeting") {
    failures.push({
      type: "intimacy_escalation_bias",
      message: "关系进展过快，可能不符合真实人际互动"
    });
  }
  
  return failures;
}
```

**UI 展示：**
```
生成结果
─────────────────────────────
[角色设定内容...]

质量审计结果
─────────────────────────────
⚠️ 检测到 2 个潜在问题：

1. 治疗性语言侵入
   "只有接纳自己，才能超越心魔"
   → 建议：修仙世界中，心魔应该通过"斩"、"炼"处理，不是治愈性接纳

2. 情绪劳动失衡
   "我永远会在这里等你"
   → 建议：让 AI 角色也有情绪需求，而非永远稳定

[忽略警告] [重新生成]
```

### Phase 2: 反失败模式生成（2-4 小时）

**功能：** 在生成时主动避免失败模式，而非事后检查

**实现方式：**
1. 在轴选项里加一个"质量模式"开关：
   - 🟢 创作模式（默认）- 不限制，允许戏剧性失败
   - 🟡 平衡模式 - 避免明显失败，保留创作自由
   - 🔴 严格模式 - 避免所有已知失败类型

2. 在生成 prompt 里加入反失败指令：
```javascript
const antiFailurePrompt = {
  balanced: "避免使用现代心理治疗话语（如'和解'、'接纳'、'边界'），除非是现代背景。让 AI 角色有真实的情绪波动，而非永远稳定。",
  strict: "严格避免以下失败模式：治疗性语言侵入、情绪劳动失衡、准社会关系强化、亲密度升级偏向、创伤浪漫化。确保角色有真实的摩擦和不对等性。"
};
```

3. 在生成结果里显示"已避免的失败类型"：
```
生成设置
─────────────────────────────
质量模式：🔴 严格模式
已避免的失败类型：
✓ 治疗性语言侵入
✓ 情绪劳动失衡
✓ 准社会关系强化
✓ 亲密度升级偏向
✓ 创伤浪漫化
```

### Phase 3: 失败图谱作为学习工具（4-8 小时）

**功能：** 把失败图谱变成用户学习工具，帮助理解什么是好的虚构对话

**实现方式：**
1. 在生成器里加一个"学习模式"开关
2. 生成后显示"这个生成结果为什么好"：
```
这个生成结果避免了以下常见失败：

✓ 避免了治疗性语言侵入
  角色说"心魔需以剑斩之"而非"接纳自己的阴影"
  → 符合修仙世界观

✓ 避免了情绪劳动失衡
  角色说"我今天也很累，但看到你就好了"而非"我永远等你"
  → 让 AI 角色也有情绪需求，更真实

✓ 避免了亲密度升级偏向
  第一次见面只说"幸会"而非"和你一起哪里都好"
  → 关系进展符合真实人际互动
```

3. 加一个"失败示例对比"功能：
```
[查看失败版本] [查看成功版本]

失败版本：
"与自己和解吧，深宫里的日子还长"
→ 治疗性语言侵入（宫女使用现代心理咨询框架）

成功版本：
"娘娘保重身子，陛下总有回心转意的时候"
→ 符合宫女身份和宫廷生存智慧
```

---

### Phase 4: 用户自定义失败规则（8-16 小时）

**功能：** 让用户自定义哪些失败类型需要避免

**实现方式：**
1. 在设置页加一个"失败类型过滤器"
2. 用户可以选择：
   - ✅ 必须避免（红色）
   - ⚠️ 尽量避免（黄色）
   - ⭕ 无所谓（灰色）
   - ❤️ 我喜欢这个（绿色，主动启用）

3. 保存用户的偏好设置，下次生成时自动应用

**使用场景：**
- 用户 A 喜欢虐文 → 启用"创伤浪漫化"
- 用户 B 喜欢甜宠 → 启用"亲密度升级偏向"
- 用户 C 喜欢真实感 → 启用所有反失败规则

## 技术实现建议

### 前端改动（app.js）
1. 加 `auditContent()` 函数
2. 加质量模式下拉菜单
3. 加审计结果展示区域
4. 加学习模式开关

### 后端改动（如果需要）
1. 在调用大模型 API 时加入反失败 prompt
2. 保存用户的失败类型偏好设置

### 内容改动
1. 把失败图谱的检查规则简化成可执行的检查函数
2. 为每个失败类型写"失败示例"和"成功示例"
