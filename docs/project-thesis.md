# Project Thesis

## 这个项目是什么

CN Failure Atlas 是一个面向虚构对话模型的失败模式分类体系，以中文场景为核心。

项目的核心贡献是识别和整理那些有经验的读者经常能感觉到、但公开评测里很少被描述清楚的失败形式——不是模型出戏、说错词、犯明显的角色知识错误，而是场景整体上是假的，即使每一部分单独看都还说得过去。

## 它要解决什么问题

公开的对话评测已经覆盖了流畅度、指令跟随、安全行为、广义人设一致性这些方面。但虚构和角色扮演对话暴露出另一类问题：模型可以写得流畅、连贯、看起来也懂情绪、表面上也符合角色——但在场景层面上仍然是失败的。信息边界先坏了导致后续分析都站不住，情绪方向读对了但动机理解错了，关系被悄悄磨平了，欲望被太早翻译了，叙事在套模板而不是在接场景，后果被提前软化了。

更重要的是，这些失败之间是有结构关系的。场景没读懂（第二层）和场景读懂了却没守住（第三层）是两种完全不同性质的问题，需要不同的干预。模型的默认写作习惯盖过场景特定需求（第四层），和前两层也不一样。把这些失败分开来看，比把所有"场景感不对"都笼统地标记为"OOC"，对诊断和改进更有帮助。

所以项目的核心问题不只是"模型能不能正确理解和续写虚构场景"，还包括：

> 有哪些反复出现的结构性问题，在让模型的输出局部看似合理的同时，整体上让有经验的读者觉得是假的？这些问题各自的失败机制是什么，如何互相区分？

## 范围

**做什么：** 中文虚构对话和角色扮演风格的交互；信息边界和世界观约束的维护；间接表达下的情绪识别和潜台词；关系逻辑驱动的场景理解；符合角色的续写和 OOC 分析；叙事结构和时间感的失真；后果处理和张力保持；多轮对话中的累积漂移。

**不做什么：** 泛化的情绪支持质量评估；通用 chatbot 安全评测；全方位 roleplay 能力打分；单纯的人设知识正确性检查；比较不同模型的总体性能排名。

## 设计原则

**命名有价值。** 读者能感觉到但说不出名字的失败仍然是真实的失败。"感觉不对"不是主观印象，是可以被分析和描述的结构性判断。给失败起名字、区分它们的边界，不只是整理词汇，而是让讨论得以发生。没有名字的失败只能被感知，不能被指认，更不容易被系统性地研究或改进。

**定义要清晰。** 每个失败类型都有清晰的定义、可观察的标准和具体示例，让"感觉不对"变成可以讨论的东西。同时，定义之间需要有明确的边界——相邻类型需要能被区分，因为混淆相邻类型会让诊断失去精度。分类体系里的边界注释（与 X 的区别是 Y）不是可选的补充，而是分类有效性的一部分。

**好看不等于真实。** 打磨过的文本不一定是真实的场景。一致的角色不一定像一个人。修复不等于后果。情感密度不等于情感质地。这个区分贯穿整个分类体系——失败的标准不是文本的表面质量，而是场景的结构完整性。有时候文笔越好、失败越难察觉，这类失败才是图谱最需要处理的核心对象。

**层级不是权重排序。** 五层对应的是失败发生的位置（前置条件、读取、保留、写作习惯、多轮），不是严重程度的排序。第一层的失败确实常常影响所有后续分析，所以有检查优先级的意义；但第四层的写作侵入失败，在密集使用场景下对体验的破坏可能和任何一层一样大。层级是诊断结构，不是重要性排名。

## 和邻近工作的关系

已有工作覆盖了广义角色扮演评测、细粒度 OOC 检测、角色知识错误识别和角色模拟 agent 训练。本项目聚焦在一个更窄的缺口：模型能不能正确读懂虚构对话中的情绪、潜台词和关系逻辑，并在续写时不引入微妙的结构性失真。
