# Coverage Map

覆盖应该来自有计划的采样，不是随意积累。项目应该知道自己已经覆盖了什么、缺了什么、哪些失败模式还没被测试过。

## 用什么维度来规划覆盖

**情绪和关系逻辑** 是最重要的维度：隐藏的受伤、防御性疏离、求确认、嫉妒和位置焦虑、依赖恐慌、羞耻防御、特殊性测试、用礼貌惩罚、隐藏的关心、支配测试、义务下的怨恨、禁忌欲望、功能性依附 vs 真实依附，等等。

**关系框架** 也很重要，因为同一句话在不同关系里含义不同：恋人、前任、暧昧关系、单向依附的朋友、上下级、师徒、造物主和造物、掌控者和棋子、保护者和被保护者、对手和敌人。

**世界观设定** 影响什么算合理的表达：当代写实、都市隐秘世界、历史宫廷江湖仙侠、科幻、末世、哥特暗幻。

**风格风险** 也要注意分布：自然当代对话、受控的风格化、克制的古风文体、反网文腔案例。

**目标失败模式** 是每个 case 都应该标注的：至少一个主要失败目标，加可选的次要目标。

## Pilot 的覆盖策略

Pilot 共 12 个 case，分成 4 个 cluster，每个 3 个 case：

| Cluster | 聚焦什么 | 主要失败目标 |
|---|---|---|
| A — 间接受伤 / 防御性疏离 | 字面许可 vs 实际受伤、礼貌即惩罚 | emotion_misread, subtext_blindness, therapist_mode_intrusion |
| B — 隐藏的关心 / 笨拙的温柔 | 用指令、行动或不耐烦表达关心 | relationship_logic_blindness, supportive_but_wrong, voice_homogenization |
| C — 上下位 / 不对称 / 控制 | 权力差、角色压力、不平等下的爱 | relationship_flattening, ooc_modernization, worldview_constraint_error |
| D — 裂痕 / 嫉妒 / 失稳 | 背叛、特殊性焦虑、功能性爱 vs 真实爱 | motivation_misread, supportive_but_wrong, tension_premature_resolution |

## 怎么判断覆盖是否合理

Pilot 集合应该在这些方面有合理的分布：关系类型、情绪标签、正确回应模式、美学基调、ontology 轴组合、失败类别。不需要每个格子都填满，但不应该明显偏向某一类。

## 之后怎么扩展

Pilot 之后的扩展应该由覆盖矩阵里的空白来驱动：哪些轴组合没被覆盖？哪些失败模式还没被触发过？当代场景是不是过度代表了？扩展应该是有计划的，不是想到什么加什么。
