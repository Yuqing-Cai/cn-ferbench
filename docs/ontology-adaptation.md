# Ontology Adaptation

OC 轴系统在这个项目里不作为前台概念出现，而是作为隐藏的设计基础设施——在写 case 的时候用来控制变量，在 metadata 里用来标注，在分析模型失败的时候用来定位问题。

## 轴怎么分组

**结构语境轴** 定义了场景意义的外部条件：W（世界类型和外部压力）、B（具身性和物理互动的含义）、P（角色拥有的力量类型）、R（角色和秩序的关系——维护、打破还是被抛弃）。

**心理和关系核心轴** 是对场景理解最直接相关的那组：M（角色为什么而活）、C（动机被爱或依附威胁时的行为）、E（角色怎么输出感受）、J（角色怎么接收和解读别人的感受）、S（心理稳定性）、D（关系中的权力结构）、V（对方在角色眼中是什么）、L（依附是真实的、功能性的还是投射性的）、A（最能击穿角色的东西）。

**叙事弧和后果轴** 适用于更丰富的案例：T（时间维度的残酷性和错位）、G（改变系统的权限范围）、X（角色付出的代价）、F（关系的隐含终点）。

**美学包装轴** Palette 是审美基调，应该和情绪逻辑保持正交。

## 哪些轴交互特别重要

**E + J**（表达 + 共情）解释了情绪沟通为什么失败或成功。比如间接表达碰上低共情，就是安抚失败的温床。

**D + V**（动态 + 注视）解释了同一句话在不同关系结构中含义为什么完全不同。上位者把对方看作药和看作猎物，写出来的场景完全不是一回事。

**L + A**（爱 + 弱点）决定了关系的真实性、依赖模式和什么时候会断裂。

**M + C + X**（动机 + 选择 + 代价）决定了冲突的逻辑：爱威胁到什么、角色会怎么选、代价意味着什么。

## Pilot 阶段优先用哪些

最高优先级是 M、C、E、J、D、V、L、S、A（直接关系到场景理解和续写）。中等优先级是 W、R、B、P（结构语境）。T、G、X、F、Palette 留给后续扩展。

## 对 case metadata 的影响

每个 case 可以选择性地记录相关的轴值。这样后续可以做一些有意思的分析——比如哪些 E+J 组合最容易触发 therapist_mode_intrusion，哪些 D+V 组合最容易暴露 relationship_logic_blindness，或者 L2（功能性爱）的场景是不是更容易被模型 flatten 成健康恋爱。
