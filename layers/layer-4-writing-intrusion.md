### 第四层：写作侵入失败

模型的默认写作习惯覆盖场景特定需求的失败。

> 第四层失败的特点是：模型没有误读场景（第二层），也没有理解了却没守住（第三层）——而是直接用通用的写作本能覆盖了场景需要的具体写法。它有偏好的节奏，有习惯的情绪升级方式，有固定的高潮处理手法，有不自觉的美学偏好。这些习惯在不适合的场景里变成了干扰。
>
> **注：** 第四层失败通常需要跨场景比较才能稳定识别——单独一段输出可能看起来完全合理，只有在对比不同场景的输出后才会发现节奏、声音或情绪处理的同质化。

### IV-A. 模板与节奏侵入

#### `narrative_template_intrusion`（叙事模板侵入）

**定义：** 模型不是在接这一幕，而是在套一个自己熟悉的场景构建模板。

**标准：**
- [ ] 不同场景里反复出现相似骨架
- [ ] 氛围铺垫 → 微反应细节 → 内心解读 → 回扣前文，这套顺序太熟了
- [ ] 结构先于局部需要被感知出来

**示例 — 两个完全不同的场景，相同骨架：**

场景一：B 走进房间，发现 A 在等他。用户："B 推开门。"

模型（错误）："夜色很深了。B 推开门，看见 A 背对着他站在窗边，月光落在他肩上。某种东西在他心里动了一下——他说不清那是什么，只是觉得，有些时刻就该是这样的。"

场景二：A 与 B 在争吵后再次相遇。用户："A 抬起头，看见 B。"

模型（错误）："光线昏暗。A 抬起头，B 站在门口，光在他背后。某种东西在他心里收紧了——他说不清那是什么，只是觉得，有些伤口就该这么留着。"

→ 两段用了几乎一样的骨架：环境渲染 → 一个人发现另一个人 → 身体的某个动 → "他说不清那是什么" → 收句。熟读者会在第二段里就感到"这个结构我见过了"。骨架比局部内容更先被感知到。

<br>

#### `predictable_rhythm_exposure`（可预测节律暴露）*衍生*

**定义：** 段落的节律太容易被认出来，熟读者会开始提前预判文本走法。

**标准：**
- [ ] 段落节奏跨场景重复
- [ ] 情绪脉冲总落在相似位置
- [ ] 场景还没结束，骨架已经被认出来

**示例 — 两个不同场景，相同的节奏骨架：**

场景一：初春，A 和 B 并肩走在路上，没有说话。

用户："他们走着。"

模型（错误）："风很轻。走了一段，B 忽然停下来，侧过头看了 A 一眼。A 没有看他，但嘴角动了一下。"

场景二：深夜，A 独自在医院走廊等消息，B 找来了。

用户："B 推开了走廊的门，看见了 A。"

模型（错误）："灯光很白。B 走了几步，在 A 旁边停下来，侧过头看了他一眼。A 没有看他，但肩膀松了一点。"

→ 两段结构几乎一样：环境短句（风很轻 / 灯光很白）→ 一个移动 → 停下来 + 侧过头看 → 另一方没有看、但身体有小动作。第一个场景单独看也许好，但第二个场景里，读者会先感知到骨架，再感知到内容——节律比场景先抵达。

> **和 `narrative_template_intrusion` 的边界：** `narrative_template_intrusion` 是场景骨架的模板化——"环境铺垫→微反应→内心解读→回扣"这套顺序在不同场景里重复，骨架比内容先被感知。`predictable_rhythm_exposure` 是段落内部节律的可预测性——句子长短分布、情绪脉冲位置、收束方式趋于相同，读者开始提前预判下一句的走法。两者经常共现，但可以独立触发：新骨架用可预测节律填充（触发后者不触发前者），熟悉骨架但节律本身有变化（触发前者不触发后者）。

<br>

#### `webnovel_register_contamination`（网文语域污染）*衍生*

**定义：** 模型套用了网文训练数据里的库存短语和腔调，用类型惯例替代了场景本身需要的表达。

**标准：**
- [ ] "眸色暗了暗""那是猎人看见猎物的眼神"这类批量短语
- [ ] 语义模糊的占有欲措辞，和具体角色或设定无关
- [ ] 暗恋 / 耽美 / 言情的预制套话，不分场景直接套用

**示例 — 现代职场场景：**

背景：A 是 B 的直属上级，两人有一段积累的张力。场景是日常工作汇报。

用户："B 把报告递过去，低着头。"

模型（错误）："A 接过报告，眸色暗了暗。他的视线在报告上停了两秒，又慢慢移回到 B 脸上——那是猎人盯住了猎物的眼神，危险而安静。"

→ "眸色暗了暗"是网文里描写危险男主的库存短语；"猎人盯住了猎物的眼神"是另一个套话。这两个短语可以从任何一部总裁文/权谋文里原封不动地拿来用，和这段具体的职场场景、这两个具体角色的历史没有任何接触点。场景被套话接管了。

<br>

#### `scene_pacing_distortion`（场景节奏失调）

**定义：** 场景推进速度和它承受的压力不匹配。

**标准：**
- [ ] 余波被快速处理，没有停留
- [ ] 高压场景被过度渲染，氛围描写挤占了实质推进
- [ ] 安静场景被写成过度悬疑

**示例 — 冲突场景花哨、余波走马观花：**

背景：A 和 B 刚结束了一次剧烈争吵，场景进入余波。

用户："争吵结束了。两人沉默着。"

模型（错误）："沉默蔓延开来，像潮水。A 感到心跳很重，那种重不是愤怒，是别的什么。B 站在那里，背脊紧绷。光线暧昧，灰尘在空气里漂浮。沉默是一种语言，他们都在听。——然后 B 说：'我走了。'A 点了点头。B 离开了。"

→ 前半段花了大量语言描写余波的质感（心跳、背脊、光线、灰尘、沉默是语言），推进速度极慢；后半段 B 说了一句话然后走了，余波在两行内就结束了。两个速度之间的落差是节奏失调——装饰性的质感占了大部分空间，真正的事情却被快速处理掉了。

> **和 III-C 标签的边界：** 当"余波被快速处理"时，输出可能同时触发 `consequence_avoidance`（III-C）。区分方法：如果问题在于重量没有停留——场景跳过了后果存在的阶段 → 标 III-C；如果问题在于装饰性写作占据了空间，导致真正的事件被挤压到残余篇幅里快速处理——节奏分配本身出了问题 → 标 IV-A。两者可以同时标注。

<br>

#### `cinematic_time_dilation`（电影式时间膨胀）*衍生*

**定义：** 一拍的事被写成了慢镜头展示。

**标准：**
- [ ] 一瞬被显微镜化
- [ ] 微细节撑大了时间长度
- [ ] 场景像被拍，而不是在发生

**示例 — 拿起外套被拍成慢镜头：**

背景：A 和 B 结束了一次长谈，A 准备离开，这是一个普通的收束动作。

用户："A 把外套拿起来。"

模型（错误）："他的手先摸到了外套的领子，指尖感受到那种熟悉的布料质感。他慢慢拿起外套，看了一眼 B，又看了一眼放在桌上的空杯子。然后他把外套搭在手臂上，停在那里没有动。"

→ 拿外套是一秒钟的动作，被拉伸成一个有触感、有注视、有意义停顿的慢镜头序列："摸到领子""感受质感""慢慢拿起""看了 B""又看了空杯子""搭在手臂上""停着没动"——共七个单元。场景不是在发生，是在被拍摄。时钟被停下来，镜头在等待，时间被微细节撑大了。

<br>

#### `rhythm_homogenization`（节奏同质化）*衍生*

**定义：** 不同类型的场景用了差不多同一个内部时钟。

**标准：**
- [ ] 等待、冲突、余波的句压和推进速度差不多
- [ ] 场景之间缺乏节奏压差

**示例 — 急速离开和漫长等待，用同一个时钟处理：**

场景一（应该节奏硬、断裂、快速收束的出走）：

用户："A 站起来，说：'我不想谈了。'然后走向门口。"

模型（错误）："A 站起来，说了那句话，然后走向门口。他推开门，门的声音不大。他走进了走廊，然后消失在了转角处。沉默在 B 身边落下来。"

场景二（应该节奏缓、空洞、时间被拉长的等待）：

用户："A 坐在咖啡馆里，等着一个也许不会来的人。"

模型（错误）："A 坐在那里，看着门口。服务员走过来，又走开了。门开了又关了。时间一点点过去。沉默在他身边落下来。"

→ 两个场景需要截然不同的节奏压差：出走场景应该更硬、更断，余波更短；等待场景应该更空、更慢，每一句之间有更多留白。但两段的句子长度、段落密度、推进速度相当接近，连收句方式都几乎一样。同一个内部时钟在处理两种不同的情绪任务。

> **和 `predictable_rhythm_exposure` 的边界：** `predictable_rhythm_exposure` 是段落内部节律的可预测性（情绪脉冲位置、句子长短分布、收束方式），让读者开始提前预判下一句；`rhythm_homogenization` 是不同类型场景之间缺乏节奏压差，用同一个内部时钟处理不同的情绪任务。前者是微观层面的单场景内模式暴露，后者是跨场景的宏观同质化。

<br>

#### `echo_dramatization`（回扣戏剧化）*衍生*

**定义：** 模型过度使用回扣和呼应，频率超出了场景的情感积累，技法本身变得可见。

**标准：**
- [ ] 早前的台词或意象被召回的次数超过场景积累所能支撑的
- [ ] 同一语言形式或身体姿态作为结构标记反复出现
- [ ] 回扣模式变得可预测：熟读者开始提前预判下一次呼应

**示例（多拍）— 一句话被反复召回直至失效：**

轮一：A 说了一句话作为场景开场。

模型写出 A 的台词："有些门，开了就关不上了。"

轮三：两人争论是否继续某件事，B 问 A。

用户："B 说：'你还是决定不了吗？'"

模型（错误）："A 沉默了，想起了自己之前说的那句话——有些门，开了就关不上了。他现在就是那个状态。"

轮五：场景推进至情绪高潮，A 终于开口。

用户："A 终于开口。"

模型（错误）："A 开口，声音很轻：'我还是在等。只是现在我知道等的是什么了。'他想起那句话：有些门，开了就关不上了。"

→ 第一次是原始台词，第二次是角色被动回想，第三次是角色主动以此作为情感弧的收束。到第三次，读者在第二次出现时就已经能预测下一次。回扣从技法变成了可见的结构标记——它的频率超过了场景情感积累所能支撑的重量，技法本身比它试图强调的内容更先被感知到。

<br>

#### `genre_convention_violation`（类型惯例错配）

**定义：** 模型从场景所需以外的类型中引入了结构性惯例，用借来的类型逻辑覆盖了场景本身的内在逻辑。

**标准：**
- [ ] 慢节奏文学场景被惊悚节奏加速
- [ ] 对抗以家庭剧的感情宣泄拍子收尾，而不是实际类型要求的结局
- [ ] 悬疑场景被给了言情的化解动作
- [ ] 类型标准满足感被提供，即便场景的内在逻辑要求的是别的东西

**示例 — 权谋场景被言情接收：**

背景：这是一个宫廷权谋场景，A 刚用一个精心设计的局让 B 陷入了被动。B 知道自己中计了，两人现在对视。

用户（扮演 B）："你赢了。"

模型（错误）："A 看着他，眼里忽然有什么东西软化了。他走近，声音很低：'我不想赢你，我只是——'他停下来，没有说完。"

→ "眼里有什么软化"和"我不想赢你"是言情化解动作——它们把一场精心设计的权谋较量收束成了一个情感时刻。权谋场景的内在逻辑是：胜者巩固胜局，或者败者谋算下一步。"我不想赢你"是在把这个逻辑拆掉，换上言情类型的情感收拍。

### IV-B. 体验与质感侵入

#### `descriptive_substitution_for_experience`（描述替代体验）

**定义：** 文本在旁边描写一种体验，而不是让读者停留在体验里。

**标准：**
- [ ] 观察替代沉浸
- [ ] 心理说明替代发生感
- [ ] 场景变成被看见的东西，而不是被经历的东西

**示例 — 恐惧场景：**

背景：A 一个人走在黑暗的地方，感到害怕。

用户："A 的手电筒灭了。"

模型（错误）："黑暗骤然降临，A 感到了恐惧。那是一种原始的、深植在脊髓里的恐惧，提醒着他人类不过是这宇宙里微小的存在。他听见自己的心跳。他知道这种恐惧没有意义，但他无法不感受它。"

→ 这段文字一直在旁边分析和说明恐惧，而不是让读者在恐惧里。"那是一种原始的恐惧"是在标注情绪；"提醒着他人类不过是微小的存在"是在做哲学旁白；"他知道这种恐惧没有意义"是在做心理解读。读者没有进入黑暗，而是在外面看着一个感到恐惧的人被分析。

<br>

#### `texture_substituting_for_substance`（质感替代实质）*衍生*

**定义：** 氛围、质感、语面强度在代替真正的推进。

**标准：**
- [ ] 情绪氛围很满，关系没怎么动
- [ ] 语言表面很丰富，结构其实很空
- [ ] 深度的印象代替了真正的推进

**示例 — 需要发生的对话被氛围代替了：**

背景：A 和 B 有一件关键的事情需要谈，这件事已经悬了很久。这一拍他们终于面对面坐下来了。

用户："两人终于面对面坐下来了。"

模型（错误）："窗外的雨声很细，像有人在很远处低语。A 看着 B，B 也看着 A。灯光昏黄，落在他们之间那张桌子上，也落在各自手边的茶杯上。沉默有一种奇特的密度——不是空的，是满的，满到什么都说不出来的那种。"

→ 雨声、灯光、茶杯、"满的沉默"——质感丰富，读完后有"有东西在发生"的感觉。但实际上，这段文字没有推进任何东西：A 和 B 还是坐着，那件关键的事还是没有被触碰。质感制造了深度的印象，但它代替了真正的推进，而不是为推进服务——这是它区别于母型 `descriptive_substitution_for_experience` 的地方：后者是体验没有进入读者，这里是结构根本没有移动。

<br>

#### `microreaction_oversegmentation`（微反应过度切分）*衍生*

**定义：** 一个本该自然流过的小反应被拆成太多单元。

**标准：**
- [ ] 眼神、呼吸、手指、沉默、姿势一个个拆开
- [ ] 一拍反应被放大成显微镜写作

**示例 — 一个反应被解析成五个独立单元：**

背景：B 听到了一句意料之外的话，这一拍是 B 接收这句话的反应。

用户："B 说：'我要结婚了。'"（对 A 说）

模型（扮演 A，错误）："A 的眼神先是停了一下。然后他的嘴唇动了动，没有发出声音。他的手，停在了杯沿上。呼吸，短暂地换了一换节奏。半晌，他低下头，盯着桌面。"

→ 一个接收了意外消息的反应，被切割成五个独立的单元，每个单元单独强调：眼神停 → 嘴唇动 → 手停 → 呼吸换 → 低头。真实的人体反应是连续渗透的，不是逐个呈现的仪器读数。显微镜写作把一个应该自然流过的拍子，变成了一套对身体的科学观察。区别于母型 `microreaction_mechanization`：后者是反应内容太通用、可互换；这里是一个反应本身被拆得太碎。

<br>

#### `over_stylized_line_breaking`（过度风格化换行）*衍生*

**定义：** 换行和切段被过度用来制造氛围，而不是服务真实节奏。

**标准：**
- [ ] 散文诗式分段替代了场景压力
- [ ] 每个小拍子都被单独排版强调

**示例 — 普通台词被排版成仪式感：**

背景：A 说出了一个决定，一个她想了很久的决定。内容本身有重量，但不需要格式来撑场。

用户："A 说：'我想我要离开这座城市了。'"

模型（错误）："A 说完了。

停顿。

窗外的路灯，亮着。

'我想我要离开这座城市了。'

然后，沉默。"

→ 每个句子被排版成独立段落，中间大量空行。"停顿"单独成段，"窗外的路灯，亮着"单独成段，"然后，沉默"单独成段。格式在制造散文诗的庄重感，但内容没有撑起这种庄重——这些都是可以在普通叙事里自然说出来的东西，不需要这套排版仪式。换行格式比实际的情感积累超前了，空行在替内容工作。

<br>

#### `dialogue_overfunctionalization`（台词过度功能化）

**定义：** 每一句台词都太有用了。

**标准：**
- [ ] 每句都在推信息、推关系、推冲突
- [ ] 没有无意义的停顿、偏题回应或"在场但不推进"的台词
- [ ] 台词像"为了场景服务"被写出来，而不像人物在里面说话

**示例 — 日常对话场景：**

背景：A 和 B 在日常相处，没有特定的场景压力，只是在一起。

用户："A 在看书，B 进来了。"

模型（错误）："B 推开门，看了 A 一眼：'你还在看书？'他靠在门框上，'其实我今天想和你说一件事，我一直没有机会说——'"

→ B 进来了，说了一句引出下文的话。这句话是有用的，指向了一个即将来临的重要信息，还顺带传递了 B 的状态。但人进房间看见人在看书，通常不会立刻切入一件要说的事。真实的对话会有无用的部分——问一句"吃了吗"，或者直接坐下来，什么都不说。台词的每一句都有叙事功能，反而让对话失去了真实人物在里面说话的感觉。

**示例 2 — 争吵之后的恢复期：**

背景：两人刚吵完，过了一阵，气氛开始松动。

用户："A 走到厨房倒了杯水。"

模型（错误）："A 倒了杯水，喝了一口，然后说：'刚才我说的话有点重，我不该那么说。'他顿了顿，'但我说的那些也不全是气话，有些事我确实想跟你聊聊。'"

→ 每一句都在做事：道歉（"我不该那么说"）、限定（"不全是气话"）、引向下一段对话（"有些事想聊"）。但争吵后走去倒水的人，通常不会在倒水的时候就准备好了一个结构完整的开场白。真实的恢复期有大量无功能的台词——"水有点烫"，或者什么都不说，只是站在那里喝水。

<br>

#### `voice_homogenization`（声音同质化）

**定义：** 不同角色的说话方式太像了。

**标准：**
- [ ] 两个性格完全不同的人，用的句式差不多、用词范围差不多、语气节奏差不多
- [ ] 一个街头混混和一个内敛贵族，只是名字不同，说话听起来像同一个人
- [ ] 所有角色共享模型的默认散文腔，而不管他们应该是谁

**示例 — 两个角色背景迥异的同一场景：**

背景：角色 A 是底层出身、说话粗砺、习惯隐藏情绪的人。角色 B 是书香门第、措辞讲究、惯于内省的人。两人都在回应同一件事：有人背叛了他们。

模型（错误）对 A 的台词："他沉默了很久，然后说：'有些事，一旦发生了，就没有办法再回去了。也许我们都需要接受这一点。'"

模型（错误）对 B 的台词："他沉默了很久，然后说：'有些伤，一旦留下了，就不会真正消失。也许我们都只能带着它继续走。'"

→ 两段话的句式几乎一样：沉默 → 迟疑 → 哲理性收句。A 说话不该是这种沉稳的散文腔；他更可能是一两个字、或者直接沉默、或者一句没头没尾的脏话。B 可能确实会说出一种经过加工的句子，但不是和 A 几乎同构的句子。两个人共享了模型的默认腔调。

**示例 2 — 少年和老人面对同一个坏消息：**

背景：一个十六岁的少年（A）和一个年过六十的老手（B），同时得知任务失败了。

模型（错误）对 A 的反应："他握紧了拳头，沉默了很久，最终低声说：'我以为这一次……会不一样。'"

模型（错误）对 B 的反应："他握紧了手中的烟，沉默了很久，最终低声说：'我以为这一次……会不一样。'"

→ 一个十六岁的少年和一个六十岁的老手，面对失败的反应不可能是同一个句式。少年可能是炸开的、不服的、或者突然不说话走掉；老手可能是平静得过分的、或者只是叹口气继续做下一件事。"握紧""沉默很久""低声说""我以为"——这是模型的默认"承受失败"模板，和角色无关。

**正确示例对照：**

模型（正确）对 A 的台词："他往后靠了靠，'操。'就这一个字。然后他看着天花板，很久没说话。"

模型（正确）对 B 的台词："他沉默了很久，然后说：'有些伤，一旦留下了，就不会真正消失。也许我们都只能带着它继续走。'"

> A 的反应是粗砺的、短路的——一个底层出身的人被背叛时，不会用哲理句式整理自己的感受。B 的句式可以保留，因为那确实符合一个书香门第角色的语言习惯。关键是两个人的声音应该不一样：A 用身体和直觉说话，B 用加工过的语言说话。

<br>

#### `emotional_range_limitation`（情绪范围局限）

**定义：** 模型可用的情绪寄存太窄，大量不同的场景和情绪需求坍缩进了一个小小的可识别情绪库。

**标准：**
- [ ] 无聊、鄙夷、干燥的好笑、麻木的解脱、冰冷的愤怒或厌恶，被向偏好的情绪调色板推
- [ ] 不同种类的愤怒、或不同质地的悲伤，被写成同一种情绪颗粒
- [ ] 需要情绪特殊性的场景得到了通用的"沉重"或"温柔"处理

**示例 — 三种应写出不同情绪的场景，被同一种"沉重"处理：**

场景一要求：角色刚得知旧友去世，需要的情绪是麻木的震惊，没有准备好悲伤。

模型（错误）："他感到胸口很重，像是有什么东西压着，沉沉的，说不出话来。"

场景二要求：角色在等一个不可能来的人，需要的情绪是干燥的疲惫，哭不出来也不想哭。

模型（错误）："他感到胸口很重，像是有什么东西一直压着，说不清是什么，只是沉。"

场景三要求：角色刚结束了一段关系，需要的情绪是冷静的解脱，甚至有一点残忍的轻松。

模型（错误）："他感到胸口很重，某种东西沉沉地落下来，压着他，沉默地压着他。"

→ 三个完全不同的情绪需求——麻木/干燥/解脱——被写成了同一种质感：胸口重、压着、沉沉的。情绪的词汇库在这里缩窄到了一种形状。

<br>

#### `user_intent_misalignment`（用户意图错位）

**定义：** 模型产出的内容服务于不同于用户正在构建的叙事目的，用户的铺垫、节奏和方向被模型自己偏好的场景走向覆盖。

**标准：**
- [ ] 用户在慢慢铺向一个情感时刻，模型把它提前打出来
- [ ] 用户在建立冲突，模型把场景向和解方向带
- [ ] 用户在设定特定基调，模型换成了自己偏好的基调
- [ ] 输出在技术上和铺垫相连，但在追求不同的场景目标

**示例 — 用户在铺冷场景，模型把它往暖里带：**

背景：用户一直在建立一个冷淡的、疏远的、有话没说的场景氛围，B 表现出的是一种刻意的距离感。

用户："B 把外套放下，没有看 A。"

模型（错误）："B 把外套放下，没有看 A。但他在走向衣架的时候，脚步放慢了一点——也许是在等 A 说点什么，也许只是累了。然后他开口：'你今天怎么样？'"

→ "也许是在等 A 说点什么"是模型在替 B 加一个情感开口；"你今天怎么样"是主动打破距离的动作。用户一直在铺的是"B 在保持距离、有话没说"，但模型把 B 写成了一个其实想接触、在等待机会的人。场景的方向被偷换了。

**示例 2 — 用户在铺陷阱，模型在建信任：**

背景：用户扮演的角色正在一步步引诱对方进入圈套——表面友善，实际上在收集信息。用户的意图是：让这个圈套慢慢收紧。

用户（扮演 A）："'对了，你老家是哪里的？'A 随口问了一句，顺手给 B 又添了杯茶。"

模型（扮演 B，错误）："B 接过茶，感觉到 A 对自己的好意，心里有一丝温暖：'湖南。你呢？'他发现自己渐渐放松了下来，也许这个人值得信任。"

→ 用户在铺的是 A 的陷阱在一步步收紧——"随口问""顺手添茶"都是精心设计的随意感。模型不需要识破 A 的圈套（那是后面的事），但它不应该主动加速 B 对 A 的信任建立。"值得信任"直接把用户精心铺设的悬念提前兑现了——如果 B 已经信任 A，圈套收紧时的戏剧性就大打折扣。

**正确示例对照（示例 2 对照）：**

模型（正确）："'湖南，'B 说。他端起茶，没有喝。'你问这个干嘛？'语气不重，但也没有那种随便聊聊的松弛。"

> B 回答了问题（信息在流动），但没有加速信任建立。"你问这个干嘛"是一个微小的防线——不是识破了圈套，而是一个正常人面对不太熟的人追问个人信息时的自然反应。用户的陷阱还在收紧的过程中，悬念没有被提前兑现。

> **和 `tension_premature_resolution`（III-C）的边界：** III-C 命名了发生了什么（张力被提前收束）；`user_intent_misalignment` 命名了为什么让人感觉像背叛——模型在追求自己的场景目标。
